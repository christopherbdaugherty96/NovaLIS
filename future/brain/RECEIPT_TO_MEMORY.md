# Receipt to Memory Policy

Defines how execution receipts may become memory or learning records.

---

## Core Principle

> Not all actions should become memory. Memory must remain useful and intentional.

---

## Memory Categories

- transient (discarded)
- session (temporary)
- persistent (approved)
- learning record (structured)

---

## Default Behavior

- receipts are generated
- receipts are not automatically persisted

---

## Promotion Rules

A receipt may become memory if:

- user explicitly approves
- it represents a reusable pattern
- it represents a failure worth remembering

---

## Learning Records

Learning records should be structured:

- domain
- action
- outcome
- lesson
- confidence
- approval status

---

## Blocked

Do not store:

- credentials
- sensitive account data
- irrelevant noise

---

## Status

Planning only
