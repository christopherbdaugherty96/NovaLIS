# Nova Second Brain Obsidian Vault - End To End

Status: future planning / vault scaffold / not runtime truth.

This document defines the complete Obsidian vault scaffold for Nova's future second brain.

The scaffold lives at:

```text
future/brain/second_brain/obsidian_vault/
```

It is intentionally non-authorizing. It does not add runtime behavior, MCP tools, dashboard code, external writes, approval behavior, or execution authority.

---

## End-To-End Flow

```text
raw capture
-> candidate proposal
-> health/lint check
-> review
-> promoted knowledge
-> rebuildable index projection
-> append-only knowledge event
-> read-only query/context
-> living dashboard visualization
```

The flow is designed so the vault can become a durable knowledge source without becoming permission.

---

## Vault Contents

```text

logs/                     second-brain operation logs, not execution receipts

indexes/                  human-readable generated indexes
VAULT_MANIFEST.md         scaffold contents and fixture rules
RUNBOOK_*.md              capture, health, and living-graph runbooks
```

---

## Hard Boundaries

```text
Knowledge is context.
Memory is not permission.
Markdown is not execution proof.
Dashboard visuals are not authority.
Ledger receipts remain proof authority.
Generated runtime docs remain runtime truth.
```

Any future runtime use must still pass through Nova's governed execution path.

## Scaffold Caveat

Example notes may use schema-valid fixture hashes.

Future strict health checks should treat those as invalid outside scaffold/test mode. Production notes must use real `sha256:` content hashes generated by the future indexer or health tool.

Before runtime promotion, future code still needs:

```text
[ ] schema loader
[ ] health/lint command
[ ] deterministic index rebuild
[ ] append-only event store
[ ] read-only query API/MCP surface
[ ] proposal/review implementation
[ ] generated runtime docs update
```
