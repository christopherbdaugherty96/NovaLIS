# Runbook - Living Graph

Status: future vault runbook / not runtime truth.

The living graph should be built only after the data layer, graph snapshot, and append-only event feed exist in runtime code.

---

## Required Inputs

```text
graph_snapshot
knowledge_event
health_summary
search_result_trace
```

The dashboard must not directly read or mutate vault files.

---

## Alive Look

```text
glowing neuron nodes
synaptic relationship edges
electric pulses along edges
idle breathing glow
new knowledge flash
review / promotion glow
reasoning cascade
conflict warning pulse
stale dimming
critical health warning pulse
```

---

## Degradation Rules

```text
cap high-cost effects
cool down per node and edge
coalesce bursts
downsample large snapshots
show redaction/truncation metadata
prefer stable graph state over animation fidelity
```

---

## Boundary

Visuals do not imply:

```text
approval
execution permission
capability certification
lock status
runtime truth
ledger proof replacement
```
