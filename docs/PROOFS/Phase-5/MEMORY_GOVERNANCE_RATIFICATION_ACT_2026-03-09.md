## Candidate Rules
1. Memory is a filing system, not a learning system.
2. No inference, no personalization drift, no behavior adaptation.
3. Memory access is explicit-invocation only.
4. Memory operations are governed and auditable (`SAVE`, `LOCK`, `DEFER`, `UNLOCK`, `DELETE`, `SUPERSEDE`).
5. Locked memory requires explicit user unlock.
6. Memory content cannot silently alter tone, authority, or execution behavior.

## Enforcement Boundary
- No runtime memory autonomy is authorized by this act.
- Any future runtime memory capability must pass Governor-mediated execution and constitutional audit.
