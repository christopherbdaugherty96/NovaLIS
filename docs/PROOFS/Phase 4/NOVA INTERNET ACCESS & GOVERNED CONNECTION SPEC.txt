
# 🌐 NOVA INTERNET ACCESS & GOVERNED CONNECTION SPEC

**Document ID:** NOVA-NET-SPEC-v1.3  
**Status:** Phase‑4 Runtime Active  
**Scope:** All outbound network traffic initiated by Nova

---

## 1. PURPOSE

This document defines the complete governance model for:

- Web search (Capability 16)
- RSS retrieval
- Any future HTTP(S) outbound request
- All internet‑based capabilities

Nova must:

> Allow explicit, read‑only intelligence access  
> Without expanding authority  
> Without hidden execution  
> Without background behavior

---

## 2. CORE PRINCIPLE

**All internet access must be:**

1. Explicitly invoked  
2. Governor‑mediated  
3. Phase‑gated  
4. Capability‑validated  
5. Single‑action constrained  
6. Logged  
7. Non‑persistent  

No exception.

---

## 3. GOVERNANCE LAYERING MODEL

Internet search follows this structural path:

```
User Input
  ↓
GovernorMediator (deterministic parsing)
  ↓
Governor (single authority choke point)
  ↓
ExecuteBoundary (phase gate)
  ↓
SingleActionQueue (concurrency gate)
  ↓
CapabilityRegistry (enable validation)
  ↓
Executor (pure, stateless)
  ↓
Network Access (via allowed mechanism)
  ↓
ActionResult
  ↓
WebSocket return
```

Executors do **not**:

- Access ledger
- Access registry
- Access boundary
- Create requests
- Self‑invoke

---

## 4. CAPABILITY 16 — WEB SEARCH

### 4.1 Invocation Rules

Valid deterministic triggers (regex‑based only, no fuzzy matching):

- `search cats`
- `search for cats`
- `look up cats`
- `research cats`

### 4.2 Execution Requirements

When invoked, **Governor must**:

1. Validate capability exists and is enabled
2. Check phase gate
3. Log `ACTION_ATTEMPTED`
4. Enforce single pending action
5. Log `SEARCH_QUERY`
6. Execute via executor
7. Log `ACTION_COMPLETED`

Failure at any stage → fail closed.

### 4.3 Executor Constraints

`WebSearchExecutor` must:

- Be stateless
- Accept only `ActionRequest`
- Return only `ActionResult`
- Perform **no** logging, capability checks, concurrency checks, persistence, or fallback routing
- Perform no summarization beyond result listing

Output must include:

- `message`
- optional `widget`
- `request_id`

### 4.4 Network Behavior

Current implementation uses `ddgs` library.  
**Future hardening goal:** All outbound HTTP must route through `NetworkMediator`.

Until then:

- Executors must **not** directly instantiate `requests` or `httpx` clients.
- Any external network library used must be explicitly documented in this spec.
- The `ddgs` library is the only currently approved exception.

`NetworkMediator` must enforce:

- Scheme restriction (HTTP/HTTPS only)
- SSRF protection (private IP, localhost, loopback blocking)
- Rate limiting per capability (see Section 10)
- Timeout enforcement
- Ledger logging of external calls

### 4.5 Non‑Authoritative Output

Internet search results are treated as **unverified external data**.  
Nova does **not** assert factual correctness beyond retrieval.  
No validation, summarization, or synthesis is performed beyond listing titles and URLs.

### 4.6 Session Containment

Internet actions must **not** inherit prior invocation context unless explicitly included in the current `ActionRequest`.  
No prior search query may influence subsequent searches.  
No cross‑session network context is permitted.

### 4.7 No Result Reordering or Ranking

Executors must return results in the order provided by the upstream source.  
No additional ranking, filtering, biasing, or heuristic modification is permitted.

### 4.8 Maximum Result Bound

Maximum search results returned per invocation: **5**.  
This limit must remain deterministic and bounded.

### 4.9 External Dependency Freeze

External network libraries used by Nova must be version‑pinned and reviewed before upgrade.  
Any behavioral change in an external dependency must be evaluated against this specification.

### 4.10 No Query Modification

The executor must transmit the user’s query exactly as received in `ActionRequest.params["query"]`.  
No automatic keyword expansion, rewriting, translation, or contextual augmentation is permitted.

## 5. RSS / TOOL NETWORK ACCESS

RSS fetch currently routes via:

```
network_mediator.request(...)
```

This is the canonical model.

Rules:

- `capability_id = None` allowed for skill bucket (shared rate‑limit pool)
- All outbound traffic logged
- No fallback search allowed
- No implicit secondary network path

If RSS fails → return empty list.  
No automatic alternate search.

---

## 6. LEDGER REQUIREMENTS

For every governed internet action:

Must log:

- `ACTION_ATTEMPTED`
- `SEARCH_QUERY` (if applicable)
- `ACTION_COMPLETED`

`NetworkMediator` must log:

- `EXTERNAL_NETWORK_CALL`
- `NETWORK_CALL_FAILED`

Ledger failure → action must fail closed.  
No silent execution.

---

## 7. PROHIBITIONS

Nova internet layer may **NOT**:

- Pre‑fetch data
- Auto‑refresh
- Poll
- Retry silently
- Switch providers automatically
- Perform background updates
- Persist search history
- Learn from search patterns
- Rank or score user intent
- Escalate to broader browsing
- Inject additional queries
- Follow links autonomously
- Scrape recursively
- Store cookies
- Maintain session memory
- Perform authentication
- Access private network
- Use alternate network stack

Explicit invocation only.

---

## 8. FAILURE BEHAVIOR

If network fails:

```
"I'm checking online. I couldn't complete the search due to a network issue."
```

If registry disabled:

```
"I can’t do that yet."
```

If phase gate disabled:

```
"That requires a specific action, which I can’t perform right now."
```

If unexpected exception:

```
"I can’t do that right now."
```

No stack traces exposed.

---

## 9. CONCURRENCY MODEL

- Only one governed action at a time
- Enforced via `SingleActionQueue`
- Cleared in `finally` block
- No parallel network calls
- No background jobs

---

## 10. SECURITY MODEL

`NetworkMediator` must:

- Block private IPs, localhost, loopback
- Validate scheme (HTTP/HTTPS only)
- Apply rate limiting:
  - **Per capability ID:** 50 requests per minute
  - **Shared bucket for tools:** 50 requests per minute (capability_id = None)
- Enforce timeout (default 5 seconds, configurable)

No DNS resolution currently performed (optional future hardening: DNS → IP validation).

---

## 11. PHASE GATE

Global control flag:

```
GOVERNED_ACTIONS_ENABLED
```

If `False`, `Governor.handle_governed_invocation` must fail **before** creating an `ActionRequest`.  
All internet capabilities must be inert. No conditional bypass allowed.

---

## 12. RETURN PATH CONTRACT

WebSocket must:

1. Send chat message first (using `action_result.message`)
2. Send widget second (if exists, as raw widget payload)
3. Send `chat_done`
4. Never send `ActionResult` object directly
5. Never access non‑existent attributes (e.g., `.user_message`)

### 12.1 Widget Schema Stability

The widget schema for search (type `"search"`, `data.results` array) must remain backward‑compatible unless versioned explicitly.  
Frontend consumers may rely on this shape.

---

## 13. FUTURE EXPANSION RULES

Future internet capabilities must:

- Be registered in `CapabilityRegistry`
- Declare `risk_level` and `data_exfiltration` flag
- Be read‑only unless explicit confirmation flow added
- Use Governor entrypoint, `ExecuteBoundary`, `SingleActionQueue`
- Log attempt + completion
- Route network through mediator

No exception.

### 13.1 Data Exfiltration Definition

Data exfiltration refers to the transmission of user‑provided content, local files, memory, or device state to an external network endpoint.  
Web search (Capability 16) transmits **only** the explicit query string provided by the user in `ActionRequest.params["query"]`.  
No local file contents, system state, or stored memory may be transmitted unless explicitly declared in the capability specification and governed by a separate confirmation flow.

---

## 14. INTENTIONAL LIMITATION

Nova search is:

- Retrieval, not browsing
- Non‑recursive
- Non‑agentic
- Non‑autonomous
- Stateless
- Read‑only

It is a governed intelligence extension, not a browsing agent.

# 🌐 NOVA PHASE-4 UPDATE

## Network Authority Unification & Free Search Migration

**Date:** 2026-02-18
**Status:** Phase-4 Runtime Active
**Scope:** Capability 16 (Web Search) + Internet Governance Layer

---

# 1. Summary

Nova has successfully completed **full network authority unification**.

The transitional `ddgs` library has been removed.
All outbound HTTP now routes exclusively through `NetworkMediator`.

Web search remains fully free, with no API keys, no scraping, and no paid services.

Architecture is now internally consistent with NOVA-NET-SPEC-v1.3.

---

# 2. What Was Removed

### ❌ Removed: `ddgs` Library

Previously:

```
WebSearchExecutor
  → ddgs
     → internal HTTP calls (unmediated)
```

This created a governance impurity:

* Hidden HTTP stack
* No mediator enforcement
* No unified rate limiting
* No centralized logging
* No SSRF enforcement
* Spec mismatch

`ddgs` has now been fully removed.

https://api.duckduckgo.com/

```

Enforced:

* Scheme validation
* SSRF protection
* Private IP blocking
* Rate limiting
* Timeout enforcement
* Ledger logging
* Capability tagging

No third-party HTTP libraries remain.
