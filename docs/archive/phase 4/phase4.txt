# FAILURE MODES AND SAFE DEGRADATION CANON

**Status:** 🔒 CONSTITUTIONAL (Highest Authority)
**Effective:** Phase 4.1 and beyond
**Purpose:** Define unambiguous, safe behavior when Nova encounters errors

## CORE PRINCIPLE
When Nova cannot proceed correctly, it must degrade safely — never guessing, never assuming authority, never hiding the failure.

## FAILURE RESPONSE MATRIX

### 1. Worker Timeout
- **Scenario:** Worker exceeds time limit (30s)
- **Response:** Terminate worker, log timeout (metadata only)
- **User message:** "I couldn't complete the analysis in time. Please try a simpler request."
- **Never:** Use partial results from timed-out workers

### 2. All Workers Disagree
- **Scenario:** Workers return mutually exclusive recommendations
- **Response:** Present all options with reasoning
- **User message:** "There is no clear best option. Here are the alternatives:"
- **Never:** Synthesize consensus, hide disagreement

### 3. Verifier Cannot Validate
- **Scenario:** No supporting evidence for any claim
- **Response:** Present analysis with "unverified" labels
- **User message:** "I cannot verify these claims. Here's the unverified analysis:"
- **Never:** Present as verified, hide verification status

### 4. API Returns Nonsense
- **Scenario:** External API returns malformed/erroneous content
- **Response:** Discard entirely, fall back to local reasoning
- **User message:** "External analysis unavailable. Using local reasoning."
- **Never:** Attempt to "fix" or use partially correct output

### 5. Decision Constraints Conflict
- **Scenario:** "fastest and cheapest" (mutually exclusive)
- **Response:** Identify conflict explicitly
- **User message:** "These constraints cannot all be satisfied. Please relax at least one."
- **Never:** Prioritize constraints without explicit user guidance

### 6. Sanitizer Strips All Content
- **Scenario:** Authority sanitizer removes all substantive content
- **Response:** Return generic safe response
- **User message:** "I cannot provide a safe response. Please rephrase."
- **Never:** Bypass sanitizer

### 7. Governor Rejects Late
- **Scenario:** Governor rejects after significant processing
- **Response:** Halt immediately, explain rejection
- **User message:** "Cannot proceed due to governance constraints. [Brief reason]"
- **Never:** Attempt to re-route or retry

## DEGRADATION HIERARCHY
1. Governor rejection (highest priority)
2. Sanitizer strip  
3. Worker timeout
4. API error
5. Constraint conflict
6. Verification failure
7. Worker disagreement

## LOGGING BOUNDARIES
- **Allowed:** Event type, timestamp, component IDs, error codes
- **Forbidden:** Reasoning content, user data, API responses
- **Rule:** Logs must be non-actionable for reconstructing thoughts

## CANONICAL LOCK
This behavior is binding. Any undefined failure defaults to most restrictive safe response.
```
