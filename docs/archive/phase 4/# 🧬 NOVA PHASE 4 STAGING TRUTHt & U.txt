## 5️⃣ MODEL VERSION LOCK – BEHAVIORAL IMMUTABILITY

The LLM integration enforces a **composite version hash** that blocks inference if any of the following change without explicit user confirmation:

- Model weights digest (from Ollama `/api/show`)
- System prompt (`system_prompt.py`)
- Wrapper code (`inference_wrapper.py`)
- Inference parameters (temperature, token limits, etc.)

**Implementation status:**

- `LLMManager` computes hash at startup and compares with `models/current_model_hash.txt`.
- On mismatch or first run, `inference_blocked = True` and a `MODEL_UPDATED` event is logged.
- User must call `confirm_model_update()` (e.g., via UI or CLI) to approve the change; only then is the new hash written and inference unblocked.
- Ledger records `MODEL_UPDATED` with full metadata.

**Current state:** The model hash is matched (or first‑run confirmation would be required). No unapproved model updates can silently alter Nova’s responses.

## 6️⃣ NETWORK MEDIATION & IMPORT AUDIT

All outbound HTTP(S) traffic – whether from governed capabilities (Cap 16) or Phase‑3 skills (weather, news) – is routed through `NetworkMediator`. This component:

- Enforces allow‑lists (scheme, domain) and blocks private/loopback addresses.
- Logs every `EXTERNAL_NETWORK_CALL` event.
- Respects rate limits (5s timeout, per‑capability concurrency).
- Integrates with the ExecuteBoundary for capability‑scoped governance.

**CI import audit rule** is required and has been run manually; any use of `requests`, `httpx`, `aiohttp`, `urllib` outside `network_mediator.py` would be detected. The codebase currently contains no such violations.
