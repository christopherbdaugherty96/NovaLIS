## 🔄 **DEFERRED EXPLAINABILITY GOVERNANCE (DEG)**

### **Core Principle: Governance ≠ Verbosity**
- **Hot Path (Fast):** Microsecond decisions, boolean outcomes, no explanations
- **Cold Path (Explainable):** On-demand explanation reconstruction
- **Rule:** "Fast default execution, silent refusals, explainable on request"

### **Architectural Split:**
```
┌─────────────────────┐    ┌──────────────────────┐
│   HOT PATH (Every   │    │  COLD PATH (Only     │
│     Request)        │    │   When Asked "Why?") │
├─────────────────────┤    ├──────────────────────┤
│ • Phase check       │    │ • Decision hash      │
│ • Capability map    │    │ • Rule ID            │
│ • Schema validation │    │ • Phase snapshot     │
│ • Hard rule match   │    │ • Reconstruction     │
│ • Boolean return    │    │ • Audit trail        │
│ • <0.1ms latency   │    │ • Can be slow        │
└─────────────────────┘    └──────────────────────┘
```

### **Hot Path Implementation:**
```python
class HotPathGovernor:
    """Fast path - submillisecond decisions"""
    
    def decide(self, request):
        # Pure, deterministic, side-effect free
        return {
            "allowed": False,
            "rule_id": "PHASE_3.5_ACTION_BLOCK",
            "decision_hash": "a9f3c2",
            # No strings, no explanations, no empathy
        }
```

### **Cold Path Implementation:**
```python
class ColdPathExplainer:
    """Explainability on demand"""
    
    def explain(self, decision_hash):
        # Reconstruct from audit trail
        audit_entry = self.audit_log.get(decision_hash)
        return {
            "decision": "denied",
            "rule": audit_entry.rule_id,
            "reason": self.rules_db[audit_entry.rule_id].description,
            "how_to_unlock": self.rules_db[audit_entry.rule_id].unlock_condition,
        }
```

### **DEG Rules:**
1. **Silent Governance:** Default refusals are brief: "I can't do that."
2. **Loud on Demand:** Full explanation only when user asks "Why?"
3. **UI ≠ Governance:** Refusal strings are UI, not governance logic
4. **Precompiled Rules:** Governance decisions are table lookups, not runtime reasoning
5. **Refusals Are Categories:** Not sentences - UI maps rule IDs to user-facing text

### **Performance Targets:**
- **Hot Path:** <0.1ms per decision
- **Cold Path:** <200ms (acceptable because user opted in)
- **Memory:** <10MB for rule tables
- **CPU:** <1% idle, <5% under load
