## IV. REQUIRED CLARIFICATIONS BEFORE PHASE-4 FINALIZATION

---

# 1️⃣ GOVERNOR MEDIATION OF ALL EXTERNAL NETWORK I/O

### Problem

Future subsystems (e.g., awareness daemons in Phase-7) may introduce background polling or independent network access unless explicitly constrained.

### Constitutional Requirement

> **All outbound network communication, regardless of origin (user invocation, daemon, subsystem, or agent), must route through the Governor and be subject to logging, rate limiting, and resource constraints.**

No module may perform direct HTTP or socket communication outside Governor mediation.

### Enforcement Strategy

* Single `NetworkMediator` under Governor authority
* Static import audit preventing raw HTTP usage outside mediator
* Ledger entry for every external call
* Rate limits enforced centrally

### Phase Impact

Phase-4 currently contains only user-invoked network calls.
This clarification future-proofs the architecture.

---

# 2️⃣ EPHEMERAL SESSION BOUNDARY DEFINITION

### Problem

Undefined conversational context lifetime may allow gradual adaptation without ratified policy change.

This risks violating:

> **Evolution vs. Optimization Principle**

### Constitutional Requirement

> **Ephemeral conversational context exists only within a single active session. A session terminates upon explicit reset, timeout, or process restart. No cross-session behavioral adaptation is permitted without a ratified policy change.**

### Operational Definition

A session ends when:

* User invokes reset
* Inactivity timeout occurs
* Application restarts
* User closes connection

Upon session termination:

* All conversational state is cleared
* No style, preference, or structural adaptation persists
* Only explicitly saved memory artifacts remain

### Prohibited

* Cross-session tone drift
* Behavioral compression
* Implicit response shortening
* Adaptive formatting

### Required Before Unlock

Session clearing must be mechanically implemented and verified.

---

# 3️⃣ PERSONALITY AGENT INFLUENCE CONTAINMENT (DESIGN-FORWARD REQUIREMENT)

### Problem

As Nova evolves toward structured reporting and proposal synthesis, influence risk increases.

While Phase-4 capabilities are mostly factual, future phases will introduce:

* Structured multi-source synthesis
* Diagnostic interpretation
* Recommendation framing

### Constitutional Position

Influence containment is not a Phase-4 runtime blocker, but must be designed before Phase-4.5.

### Design Directive

A future **Style Constraint Layer** must:

* Flag prescriptive verbs ("should", "must", "recommend")
* Flag emotional framing
* Flag directive phrasing
* Avoid persuasive narrative construction

This layer may annotate but must not censor output unless policy violation occurs.

### Status

Design required before Phase-4.5.
Not required to unlock Phase-4 runtime.
