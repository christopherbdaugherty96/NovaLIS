# NovaLIS — Canonical Project Roadmap & Status

**(Governor-First Local AI System)**

---

## 1. What NovaLIS *Is* (Read This First)

NovaLIS is **not** an AI assistant that acts on its own.

NovaLIS is a **local, sovereign AI Governor** whose job is to:

> **Control, constrain, and mediate intelligence — not replace human authority.**

Core philosophy:

* Intelligence may **suggest**
* NovaLIS may **propose**
* **Only the user confirms**
* Execution is **deterministic, explicit, and gated**

There is **no autonomy**, **no background behavior**, and **no learning** in Phases 1–3.

---

## 2. Non-Negotiable System Invariants (LOCKED)

These apply to *every phase unless explicitly unlocked*.

### Authority & Safety

* LLM output **can never execute**
* LLM output **can never create intent**
* LLM output **can never bypass confirmation**
* No multi-action chaining
* One action at a time, always

### Behavior

* Silence-first (no follow-ups unless asked)
* Fail closed (errors → do nothing)
* No retries, no self-correction loops
* No background polling, no schedulers, no “smart” behavior

### Architecture

* Governor (brain_server) is the **sole authority**
* Execution only via explicit `ActionRequest`
* Confirmation only via `ConfirmationGate`
* Executors are deterministic and allow-listed

## 6. Phase 2.1 — What Is Already DONE (LOCKED)

### ✅ GovernorMediator (Minimal, Correct)

* Intercepts **all** LLM output
* Validates origin
* Requires user-initiated context
* Strips authority / execution language
* Detects multi-step / chaining language
* Hard-stops on violations
* Returns **inert text only**
* No routing
* No intent inference
* No ActionRequest creation

### ✅ SingleActionQueue

* Enforces one pending action
* Clears on hard stop
* No chaining possible

### ✅ ActionRequest & ConfirmationGate

* Reviewed and approved
* Immutable
* ConfirmationGate is sole resolver
* Metadata fields allowed but non-operative

### ✅ Roadmap & Scope

* Phase 1 & 2 frozen
* Phase 3 blocked
* No UI or feature work allowed
