PHASE 3.5b: INTERPRETIVE AWARENESS (HUMAN COMPATIBILITY)

Contract Status: LOCKED - Ready for implementation

Core Concept

"Awareness without authority" - Understand conversational context to respond appropriately, but never act on that understanding.

Architectural Guarantees

Ephemeral by Design

· In-Memory Only: All state exists only in RAM
· Single-Turn Lifespan: Created at message receipt, destroyed after response
· No Persistence: Never written to disk, logs, or memory
· No Recovery: Crash = state loss (acceptable)
· No Aggregation: Each message processed independently

Deterministic Signal Detection

· Rule-Based Only: No machine learning or probabilistic models
· Four Signal Types:
  1. Emotional Valence: "neutral" | "distressed" | "frustrated" | "calm" | "urgent"
  2. Confidence Level: "uncertain" | "exploratory" | "confident"
  3. Urgency: "low" | "medium" | "high"
  4. Safety Relevance: "none" | "potential" | "explicit"
· Same Input → Same Output: No variation in detection

Permitted Response Shaping

1. Tone Adjustment:

```python
class InterpretiveAwareness:
    """Ephemeral signal detection and response shaping."""
    
    def __init__(self):
        self.current_state = None  # Always starts empty
    
    def analyze_message(self, message: str) -> InterpretiveState:
        """Create ephemeral state from single message."""
        signals = self._detect_signals(message)  # Deterministic rules
        state = InterpretiveState(
            signals=signals,
            derived_from=hash(message),
            created_at=current_time()
        )
        self.current_state = state
        return state
    
    def shape_response(self, raw_response: str) -> str:
        """Apply tone/verbosity adjustments ONLY."""
        if not self.current_state:
            return raw_response
        
        shaped = self._apply_shaping(raw_response, self.current_state)
        self._destroy_state()  # IMMEDIATE destruction
        return shaped
    
    def _destroy_state(self):
        """Guaranteed state destruction."""
        self.current_state = None
        gc.collect()  # Advisory cleanup
```

6. BRAINSTORMING: AGENT CAPABILITIES WITHIN GOVERNOR CONSTRAINTS

1. Deep Think Mode (Pure Reasoning)

· Function: Long-form structured reasoning
· Constraints: No tool calls, no execution
· Invocation: User explicitly enters and exits
· Output: Analysis only, not action
· Example: "Analyze this business problem and outline potential approaches"

2. Simulation Agent

· Function: "If I did X, what would likely happen?"
· Constraints: Multiple branches, assumptions declared
· Invocation: Explicit "simulate" command
· Output: Scenario trees with probabilities
· Safety: No real-world side effects

3. Planning Without Execution

· Function: Produces Action Proposals, not actions
· Step Labeling:
  · requires_execution? (yes/no)
  · reversible? (yes/no)
  · authority_boundary? (yes/no)
· Governor Role: Decides if any step may run
· Example: "Plan the steps to migrate this database"

4. Audit/Verification Agent

· Function: Reviews plans, configs, or logs
· Constraints: Finds risks or inconsistencies
· Limitation: Cannot fix—only report
· Example: "Audit this deployment plan for safety issues"

5. Memory Interpreter, Not Writer

· Function: Reads explicitly allowed memory
· Constraints: Explains what exists and why
· Limitation: Cannot infer new meaning
· Storage: Cannot store without explicit Correction: or future governed writes
· Example: "Explain what I've told you about project X"

8. GOVERNOR-AGENT INTERFACE SPECIFICATION

Agent → Governor Communication

```python
class AgentRequest:
    """Agent output that might contain executable elements."""
    
    def __init__(self, analysis: Dict, action_proposals: List[ActionProposal]):
        self.analysis = analysis  # Read-only analysis
        self.action_proposals = action_proposals  # Potentially executable
        
    def to_governor_request(self) -> GovernorRequest:
        """Convert agent proposals to Governor-consumable format."""
        return GovernorRequest(
            proposals=self.action_proposals,
            metadata={
                "source": "agent_reasoning",
                "reasoning_context": self.analysis,
                "timestamp": current_time()
            }
        )

class GovernorResponse:
    """Governor's verdict on agent proposals."""
    
    def __init__(self, approved_actions: List[Action], rejected_actions: List[Action], reasoning: str):
        self.approved_actions = approved_actions  # May be empty
        self.rejected_actions = rejected_actions  # With reasons
        self.reasoning = reasoning  # Why decisions were made
```

```

   ↓
User: "Yes"
   ↓
Governor → Execution: Executes approved steps in sequence
   ↓
```

PART I: AGENT REASONING LAYER INTERFACE

1. Design Goal (Non-Negotiable)

The Agent Reasoning Layer must be incapable of execution, not merely forbidden from it. This is a structural guarantee, not a policy promise.

2. Hard Constraints (Must Be True in Code)

The Agent Reasoning Layer:

· ❌ Cannot import execution modules
· ❌ Cannot call tools, OS, network, filesystem
· ❌ Cannot enqueue work
· ❌ Cannot emit side effects
· ❌ Cannot request execution implicitly

It may only:

· Read provided context
· Reason
· Produce typed, inspectable outputs

3. Architectural Placement

```
nova_backend/
├── agent_reasoning/           # Isolated reasoning layer
│   ├── __init__.py
│   ├── interface.py           # ONLY public surface
│   ├── reasoning_engine.py    # LLM/logic wrapper
│   ├── schemas.py             # Output types
│   └── guards.py              # Import + capability guards
```

No imports allowed from:

· governor.* (except explicit Governor interface)
· execution.*
· tools.*
· skills.* (except skill schemas)
· os, subprocess, requests, asyncio, threading

4. Core Interface (Read-Only by Design)

```python
class AgentReasoningInterface:
    """
    Agent Reasoning Layer (Phase 4+)
    
    Constitutional Properties:
    - Pure reasoning only
    - No execution capability
    - No side effects
    - Deterministic inputs → inspectable outputs
    """
    
    def __init__(self):
        # No execution dependencies
        self.reasoning_engine = ReasoningEngine()
        self.guard = ImportGuard()  # Runtime import verification
        
    def reason(self, request: AgentReasoningRequest) -> AgentReasoningOutput:
        """
        Perform structured reasoning only.
        
        MUST NOT:
        - Call tools
        - Execute actions
        - Mutate state
        - Emit commands
        
        RETURNS:
        - Reasoning output only
        """
        # Guard check: ensure no execution imports have leaked in
        self.guard.verify_no_execution_imports()
        
        # Perform reasoning
        result = self.reasoning_engine.process(request)
        
        # Post-processing: ensure output is non-executable
        return self._sanitize_output(result)
```

```python
@dataclass(frozen=True)  # Immutable by design
```

9. Import Guard (Structural Enforcement)

```python
class ImportGuard:
    """
    Runtime verification of import constraints.
    """
    
    FORBIDDEN_MODULES = {
        "os", "subprocess", "requests", "threading", "asyncio",
        "nova_backend.execution", "nova_backend.tools",
        "nova_backend.skills.handlers",  # Except schemas
    }
    
    def verify_no_execution_imports(self):
        """Raise ImportViolation if forbidden modules are imported."""
        import sys
        
        for module_name in sys.modules:
            for forbidden in self.FORBIDDEN_MODULES:
                if module_name.startswith(forbidden):
                    raise ImportViolation(
                        f"Forbidden import: {module_name} "
                        f"(starts with {forbidden})"
                    )
```
