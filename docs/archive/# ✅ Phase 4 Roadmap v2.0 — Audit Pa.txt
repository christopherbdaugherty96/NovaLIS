## 🔹 Executive Summary

Phase 4 introduces **governed execution** – the ability for Nova to perform explicit, user‑invoked actions with real‑world effects. All execution remains:
- **Explicitly invoked** – no autonomy, no background, no prediction
- **Governor‑mediated** – every action passes through the single choke point
- **Auditable** – every attempt recorded in the immutable ledger
- **Constitutionally constrained** – all Tier 1 invariants remain absolute

This extended roadmap defines a **three‑tier capability ladder** that progressively increases Nova’s perceived intelligence **without expanding authority**:

| Phase | Focus | Key Capabilities | Authority Level |
|-------|-------|------------------|-----------------|
| **4** | Anchor Layer – Trusted Research | Governed Web Search, Structured Research Mode, Source Transparency | Read‑only, explicit invocation |
| **4.2** | Cognitive Depth – Expert Analyst | Multi‑step tool chaining, Deep Analysis Mode, Report Formatting Engine | Read‑only, explicit invocation |
| **4.5** | Household Intelligence – Advisor | System Diagnostics, Network Analysis, Device Status (read‑only) | Read‑only, explicit invocation |

All capabilities remain **explicitly invoked**, **Governor‑mediated**, and **read‑only**. No autonomy, no background work, no predictive systems.

## 🏛️ Architecture Overview (Unchanged)

All execution flows must integrate with the existing Governor components. The **single choke point** is `governor.py` – it is the only file that creates `ActionRequest` and calls `ExecuteBoundary`. `governor_mediator.py` is purely for input parsing and invocation detection; it may **not** create action requests directly.

```
User Input → WebSocket → governor_mediator.mediate()
    → (if execution requested) → governor.py (SOLE CHOKE POINT)
        → CapabilityRegistry (pre‑action check with absolute path resolution)
        → ExecuteBoundary (phase gate, resource limits)
            → ConfirmationGate (if required; tokens are cryptographically random UUIDv4, invalidated on session end)
                → Executor (dumb, bounded) → uses NetworkMediator for any I/O
                    → ActionResult → Ledger → UI
```

All outbound network communication must pass through **NetworkMediator** (under Governor authority). No other module may import `requests`, `httpx`, `urllib`, `socket`, or similar. **A CI import audit enforces this rule.**

## 🔐 Model Version Lock – Per‑Inference Validation

The model version lock system (introduced as a Phase‑4 safeguard) now includes **per‑invocation validation**:

- The composite hash (model digest + system prompt + wrapper code + parameters) is recomputed and compared against the stored hash **before every inference call**.
- If a mismatch is detected, inference is blocked and a `MODEL_UPDATED` event is logged (if not already confirmed).
- Headless mode also blocks inference – no auto‑approval.

This ensures that any runtime change to the model, prompt, wrapper, or parameters is detected immediately and cannot be used without explicit user confirmation.

---

## 🔐 Confirmation Tokens – Cryptographic Randomness

All confirmation tokens generated by `ConfirmationGate` are:

- Cryptographically random (UUIDv4 or `secrets.token_urlsafe(32)`)
- Bound to a session ID
- Expire at session end (or after a timeout)
- Stored only in memory, never persisted

This prevents token guessing, replay, or cross‑session reuse.

## ⚠️ Risk Zones & Mitigations (Adversarially Verified)

| Risk | Mitigation |
|------|------------|
| **Bypass of governor.py** | Static analysis ensures only governor.py creates ActionRequests; governor_mediator.py forwards only. |
| **Registry path fragility** | Absolute path resolution relative to module; test with different cwd. |
| **Registry unavailability** | Explicit fail‑closed: if registry missing/malformed, no actions proceed. |
| **Split authority** | governor.py is sole choke point; all execution flows converge there. |
| **Direct network calls outside NetworkMediator** | CI import audit bans `requests`, etc., outside `network_mediator.py`. |
| **Model version drift** | Version hash includes weights, prompt, wrapper, parameters; **validated on every inference call**; update requires confirmation; headless mode blocks inference. |
| **Confirmation token reuse across sessions** | Tokens are cryptographically random, bound to session ID, invalidated on session end. |
| **Session persistence** | Session manager explicitly clears all conversational state, corrections, pending tokens. |
| **Environment variable override** | `GOVERNED_ACTIONS_ENABLED` is hard‑coded in `execute_boundary.py`; no env reads. |
| **Silent fallback** | All fallback paths require user confirmation; code audit ensures none exist. |
| **Ledger write failure** | If ledger write fails, action execution is denied; no action proceeds unlogged. |

## 🔮 After Phase 4: Phase 4.2 and Phase 4.5

Once Phase 4 runtime is stable for ≥30 days, work on Phase 4.2 (Cognitive Depth) and Phase 4.5 (Household Intelligence) can begin. These phases add **no new authority** – only more sophisticated use of existing read‑only capabilities. Each new capability must pass its own code review and adversarial testing, but does not require a second global unlock.

**Phase 4.2** focuses on multi‑step reasoning and structured output.  
**Phase 4.5** focuses on environment awareness (system, network, device).

Together, they transform Nova from a trusted research engine into an expert analyst and household intelligence advisor – all without autonomy or background behavior.
