# Product Legibility

Nova has entered a new priority phase:
Not just adding backend depth, but making existing depth understandable, usable, and credible at first touch.

## Goal
A new user should understand why Nova matters within 60 seconds.

## What Product Legibility Means

## 1. Visible Governance
Users should see how decisions are made.

Examples:
- action requested
- capability used
- confirmation required
- blocked reason
- result

## 2. Immediate Competence
A user should succeed quickly without reading extensive docs.

## 3. Clear Boundaries
Users should understand:
- what Nova can do
- what Nova will not do
- why

## 4. Proof of Reality
Use:
- screenshots
- real flows
- real commands
- honest limitations

## 5. Confidence
Nova should feel deliberate, stable, and trustworthy.

## Anti-Pattern
Do not hide weak UX behind architecture language.

## Current Priority Outputs
- Trust Review Card
- Starter commands
- Screenshot proof
- Better first-run path
- Demo flow
