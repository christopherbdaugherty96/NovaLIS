# Why Nova

Nova exists because useful AI should not require unchecked authority.

## The Problem
Many AI assistants are helpful, but their action model is often unclear:
- What did the assistant understand?
- What can it actually do?
- What did it execute?
- What did it refuse?
- What did it log?
- Who remains in control?

Nova is built around making those boundaries visible.

## Nova Compared To Generic Assistants

| Capability Area | Generic AI Assistant | NovaLIS |
|---|---|---|
| Helpful answers | Yes | Yes |
| Local-first direction | Sometimes | Core direction |
| Visible action boundaries | Often limited | Core design goal |
| Approval before sensitive action | Inconsistent | Required for governed actions |
| Runtime truth docs | Rare | Explicit generated docs |
| Capability registry | Usually hidden | Central control surface |
| Ledger / action history | Usually opaque | Core governance surface |
| Broad autonomy | Often a selling point | Intentionally limited |

## Core Difference
Nova does not treat intelligence as authority.

A model may help reason, summarize, draft, and suggest. Real actions must remain bounded, inspectable, and governed.

## Best Fit Today
Nova is best for users who value:
- local control
- transparent behavior
- reviewable actions
- useful daily assistance
- trust over maximum autonomy

## Honest Limit
Nova is still an active alpha build. Its strongest long-term value is the governance model, but the product experience is still being finished.
