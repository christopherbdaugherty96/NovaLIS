# Demo Preview Excellence Standard (2026-04-23)

## Purpose
A demo should not only function.
It should create trust, clarity, and excitement within minutes.

## Core Standard
Every demo should feel:
- stable
- clear
- valuable
- premium
- memorable

## First Impression Flow
Within the first two minutes, a user should be able to:
1. Understand what Nova is
2. Ask something useful
3. See a strong response
4. Understand visible boundaries
5. Want to continue exploring

## Reliability Standard
- no obvious errors
- no transcript confusion
- no delayed or ghost responses
- clear loading states
- smooth navigation

## Presentation Standard
- strong visual hierarchy
- calm spacing
- polished copy
- centerpiece feels intentional
- no clutter

## Demo Flows To Keep Ready
### Flow 1 — Research
Ask a real question and receive a useful structured answer.

### Flow 2 — Local Utility
Use a bounded local action or system insight.

### Flow 3 — Governance Trust
Show visible limits, confirmations, or runtime truth.

### Flow 4 — Continuity
Show memory or follow-up context in a controlled way.

## Demo Operator Rule
Do not overload a demo with every feature.
Show only the strongest paths.

## Pass Condition
A normal user should leave thinking:
This feels real, useful, and different.
