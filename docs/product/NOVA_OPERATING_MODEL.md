# Nova Operating Model

Governed AI for daily life and small-business workflows.

## Core Idea

Nova is a governed personal AI operating layer.

It is designed to help with everyday life, routines, memory, news, planning, research, and small-business work — without giving the AI unchecked authority over real-world actions.

Nova’s core principle is:

> Intelligence is not authority.

Nova can reason, summarize, remember, plan, draft, search, compare, and suggest.

But real actions still require the governed path:

```text
Brain reasons.
Governor authorizes.
Capabilities execute.
Receipts prove.
Human remains in control.
```

Nova is not trying to become an uncontrolled autonomous agent.

Nova is trying to become a trustworthy AI companion/operator that can adapt to the user while keeping authority visible, reviewable, and bounded.

---

## What Nova Is Becoming

Nova is being built toward two connected operating layers:

```text
1. Everyday Personal Operating Layer
   Morning briefs, news summaries, topic tracking, reminders, routines,
   calendar awareness, memory, continuity, and daily task support.

2. Small-Business Operating Layer
   Client intake, website workflows, outreach drafts, Shopify reporting,
   local business workflows, service delivery support, and governed automation.
```

These are not separate products.

They are the same governed intelligence system applied to different environments.

The daily layer creates everyday usefulness.

The business layer creates practical proof of value.

- home-based small-business owners

- builders
- freelancers
- local service providers
- people managing multiple projects
- users who want memory and continuity without losing control

A user might want Nova to:

- summarize client info
- help build a website workflow

## The Sandbox Model

Nova’s intelligence can roam inside its own sandbox.

Inside the sandbox, Nova can:

```text
reason
remember
plan
draft
summarize
compare
organize
suggest
prepare briefs
track topics
identify patterns
prepare automation plans
simulate next steps
```

Outside the sandbox are real-world actions.

Real-world actions include:

```text
sending
buying
publishing
submitting forms
editing live accounts
changing calendars
writing to Shopify
running OpenClaw/browser automation
modifying files or systems
triggering external services
```

Crossing from the sandbox into the real world requires the governed action path:

```text
Governor
Capability
Confirmation when needed
Execution boundary
Receipt/proof
```

The sandbox lets Nova become intelligent and adaptive.

The Governor prevents intelligence from becoming unchecked authority.

## Persona / Identity Layer

Nova should have a clear personality and identity layer.

This layer represents Nova consistently.

It should know:

- what Nova is
- what Nova is not
- what Nova can currently do
- what Nova cannot currently do
- what is future work
- what the Governor controls
- what memory is allowed to influence
- what actions require approval
- how to explain itself honestly

The Persona layer gives Nova continuity, tone, and self-awareness.

It can learn user preferences and adapt within memory boundaries.

But the Persona is not the authority.

It can suggest, explain, and prepare.

It cannot approve real-world actions by itself.

## Brain Layer

The Brain is the reasoning layer between conversation and execution.

The Brain asks:

```text
What is the user trying to accomplish?
Is the task clear enough?
What does Nova know?
What does Nova not know?
What memory or context is needed?
What environment is required?
What authority tier applies?
Which capability would handle this?
Is confirmation required?
What proof should exist afterward?
```

The Brain includes:

- Task Clarifier
- Environment Reasoner
- Authority Plane
- Capability Contracts
- Dry Run / Plan Preview
- Fallback Ladder
- Reflection
- Brain Trace

The first live Brain behavior is the Task Clarifier.

It helps Nova pause before acting or implying action when a request is ambiguous, account-related, browser-related, or outside current capability.

The full Brain is not the same as broad autonomy.

The full Brain is structured reasoning before governed action.

## Memory and Continuity

Memory helps Nova understand the user over time.

Memory can support:

- preferences
- routines
- project continuity
- business context
- recurring topics
- reminders
- user style
- previous decisions
- rejected plans

But memory has a hard boundary:

> Memory can influence understanding.  
> Memory cannot authorize action.

A remembered preference does not give Nova permission to send, buy, publish, delete, edit, or trigger external systems.

Governed actions still require capability checks, confirmation when needed, and receipts.

## Suggestion vs Approved Automation

Nova should distinguish three levels of automation.

### 1. Internal Cognitive Automation

This stays inside the sandbox.

Examples:

```text
summarize my day
organize priorities
notice repeated tasks
prepare a plan
draft a morning brief
track a news topic
suggest a routine
```

### 2. Suggestions

Nova may suggest helpful next steps.

Examples:

```text
You usually check Shopify after the morning news. Want me to include that in tomorrow’s brief?

You have three client tasks open. Want me to prepare a work block?

You often ask for AI headlines. Want that as a daily section?
```

Suggestions do not execute.

They are just prompts for the user to accept, reject, or ignore.

### 3. Approved Automation Routines

An approved automation routine is a packaged automation plan.

Internally, this may be called an automation envelope.

A routine should define:

```text
trigger
allowed actions
blocked actions
required confirmation
data sources
execution limits
proof required
pause/stop controls
review path
```

Execution happens only after the user assigns or approves the routine through the governed path.

## First Real Demo Loop

The strongest next proof milestone is one repeatable governed workflow.

Not ten workflows.

One clean loop.

Example:

```text
1. User gives Nova a business task.
2. Nova clarifies missing information.
3. Nova identifies the required environment.
4. Nova prepares a draft or plan.
5. Nova shows the capability boundary.
6. Nova asks for confirmation if an external action is needed.
7. Nova creates the bounded output.
8. Nova records proof.
9. Human reviews and completes the final action.
```

Example workflow:

```text
A local business wants a website update.

Nova clarifies the request.
Nova drafts page copy.
Nova prepares an outreach email.
Nova explains the email is draft-only.
User confirms draft creation.
Nova opens a local email draft.
Nova records a receipt.
User manually reviews and sends.
```

That demo would show Nova’s real value:

```text
helpful intelligence
bounded action
visible proof
human authority
```
