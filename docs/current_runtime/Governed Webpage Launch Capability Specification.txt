# 📘 NOVA — Governed Webpage Launch Capability Specification

**Document ID:** `NOVA-PHASE4-WEB-LAUNCH-v1.0`
**Status:** Phase-4 Capability Proposal
**Scope:** Explicit Webpage Opening (Non-Search)
**Authority Level:** Governed Execution (User-Invoked Only)
**Classification:** Constitutional / Technical / Product

---

# 🔹 1. Purpose

This document defines how Nova may open web pages in a governed, explicit, and non-autonomous manner.

This capability:

* Does not perform search by default
* Does not infer intent
* Does not auto-navigate
* Does not persist browsing behavior
* Does not execute background actions

It allows:

> Explicit user-invoked opening of predefined or user-confirmed web domains.

This is an execution capability and therefore must be strictly Governor-mediated.

---

# 🔹 2. Core Principle

Opening a webpage is an execution action.

Execution always requires:

* Explicit invocation
* Governor validation
* Phase authorization
* Deterministic mapping
* Clear target domain

No guesswork.
No silent selection.
No background behavior.

---

# 🔹 3. Capability Classification

**Type:** Governed Action
**Execution Scope:** Local browser launch only
**Authority Level:** Low-risk execution
**Persistence:** None
**Autonomy:** Prohibited

# 🔹 5. Trigger Architecture

## 5.1 Preset Open Commands

Trigger examples:


```
open Youtube
open Google
open Facebook
open Pandora
open GitHub
open Twitter
```

Mapping:

* Deterministic
* Exact or phrase-cluster based
* No ranking
* No inference

If match found in preset registry:

→ Proceed to Governor validation
→ Launch associated URL

---

## 5.2 Non-Preset Domain Flow

If user says:

* “Open Medium”
* “Open Spotify”
* “Open example.com”

And domain is not in preset list:

### Flow:

1. Perform governed lookup to identify primary domain.
2. Present candidate URL.
3. Ask explicit confirmation:

> “I found [https://www.medium.com](https://www.medium.com). Open this site?”

4. Only after explicit “Yes” does execution occur.

If user declines:

→ No action taken.

No auto-open.
No guessing.
No multiple candidate selection without confirmation.

---

# 🔹 6. Governor Mediation Requirements

Before execution, Governor must validate:

* Action is user-invoked
* Domain string is explicit
* No URL manipulation
* No query string injection
* No redirection attempt
* No executable payload
* Phase-4 execution enabled

If validation fails:

→ Deny action.

---

# 🔹 7. URL Safety Constraints

The system must:

* Strip query parameters unless explicitly requested
* Block javascript: URIs
* Block file:// access
* Block localhost unless explicitly allowed in future phase
* Block direct IP navigation unless confirmed
* Block download links
* Block non-HTTP/HTTPS protocols

Allowed protocols:

* https://
* http:// (discouraged, may warn)

---

# 🔹 8. Execution Behavior

When approved:

* Open URL in default system browser
* Do not inject automation
* Do not simulate clicks
* Do not auto-fill forms
* Do not track behavior
* Do not remain attached to browsing session

Nova opens and disengages.

---

# 🔹 9. Confirmation Model (Non-Preset Only)

For non-preset domains:

Nova must:

* Display exact URL
* Ask one confirmation question
* Require explicit “Yes” or equivalent deterministic approval

No implicit confirmation.

If ambiguous:

> “Did you want to open the official site at https://…?”

Single clarification only.

---

# 🔹 10. Audit Logging

Each webpage launch logs:

```
{
  timestamp,
  trigger_type: preset | dynamic,
  requested_name,
  resolved_url,
  confirmation_required: true/false,
  confirmation_received: true/false,
  execution_result: success/failure
}
```

Logs are:

* Local
* Immutable
* Not user-visible by default
* Auditable

---

# 🔹 11. Failure Modes

| Scenario                   | Behavior                  |
| -------------------------- | ------------------------- |
| Domain not found           | Inform user, no execution |
| Multiple candidate domains | Ask clarification         |
| Timeout during lookup      | Inform user               |
| Confirmation denied        | Abort                     |
| Browser launch fails       | Inform user               |

No retries.
No background continuation.

---

# 🔹 12. Guardrails

This capability must not:

* Trigger web intelligence automatically
* Suggest alternative sites
* Auto-complete URLs
* Save browsing history
* Learn preferred sites
* Expand preset registry autonomously
* Remember previously confirmed domains (Phase-5+ only)
* Schedule website opening

No autonomy.
No learning.
No drift.
