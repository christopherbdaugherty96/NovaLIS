## 🔹 Executive Summary

Phase 4 introduces **governed execution** – explicit, user‑invoked actions with real‑world effects. The design is constitutionally sound; the runtime remains locked until a verified admission gate is passed.  

The meta‑review identified a core tension: **constitutional perfection vs. buildable reality**. The resolution is to add four lightweight, high‑impact safeguards **now** – as implementation details, not constitutional amendments – and to defer all speculative future risks to later phases.  

This plan updates the Phase 4 Roadmap with those safeguards, aligns them with existing workstreams, and provides a consolidated admission checklist. **The goal is to ship Phase 4 on a Raspberry Pi, not to write more laws.**

---

## ✅ Four Immediate Safeguards (Integrate into Phase 4)

| # | Safeguard | Why Now | How | Roadmap Integration |
|---|-----------|---------|-----|---------------------|
| 1 | **Capability Registry & Lifecycle** | Track, version, and revoke capabilities. Essential as execution surface grows. | Static `registry.json` (or DB) with `id`, `risk_level`, `permissions`, `enabled`. Governor checks every action. Add capability installation as logged user event. | Workstream C (Capabilities) – extend registry task. |
| 2 | **Model Version Lock** | Prevent silent behavior changes from model/API updates. | Each model (local or external) identified by version hash. Any update requires `MODEL_UPDATED` ledger event + user confirmation. | Workstream B (Execution Layer) – add version check to executor calls. |
| 3 | **External LLM as Governed Capability** | Data leaving home must be transparent and user‑controlled. | Register external LLM calls as capabilities with `data_exfiltration` flag. Governor checks user consent for service. Log metadata (not content) in ledger. | Workstream C – wrap external APIs with governed wrapper. |
| 4 | **Device Authority Scopes** | Granular permission levels per domain (observe, suggest, confirm, automatic). | Extend policy engine with `authority_level`. Design now, implement in Phase 4.5. Display in Trust Panel. | Phase 4.5 design task (add to roadmap). |

**These four items are low‑effort, high‑impact, and fully aligned with the existing constitutional framework.** They do not require new Tier‑1 invariants – they are disciplined implementation.
