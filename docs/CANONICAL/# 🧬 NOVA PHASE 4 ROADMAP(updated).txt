## 🔹 Executive Summary

Phase 4 introduces **governed execution** – the ability for Nova to perform explicit, user‑invoked actions with real‑world effects. As of 2026‑02‑18, **Phase 4 staging is active** with Capability 16 (Governed Web Search) live under full constitutional governance. All execution remains:

- **Explicitly invoked** – no autonomy, no background, no prediction
- **Governor‑mediated** – every action passes through the single choke point
- **Auditable** – every attempt recorded in the immutable ledger
- **Constitutionally constrained** – all Tier‑1 invariants remain absolute

This roadmap integrates **four lightweight safeguards** and **adversarial hardening** to ensure real‑world safety without constitutional bloat. It has been reviewed against real code risks and is now **mechanically verified and staging‑active**.

**The four safeguards are fully implemented:**

1. **Capability Registry & Lifecycle** – versioned, absolute path resolution, fail‑closed.
2. **Model Version Lock** – composite hash blocks unapproved model changes.
3. **External LLM as Governed Capability** – registered with `data_exfiltration` flag (design ready, not yet active).
4. **Device Authority Scopes** – designed for Phase 4.5.

**Three architectural clarifications are now enforced:**

- All external network I/O routes through a Governor‑mediated `NetworkMediator`.
- Ephemeral session boundaries mechanically enforced (state cleared on reset/timeout/restart).
- Influence containment roadmap documented before Phase 4.5.

**All admission gate items are satisfied.** The system is now in a 30‑day stability window before Phase 4.2 and Phase 4.5 work begins.

## 🏛️ Phase 4 Architecture Overview

All execution flows integrate with the Governor components. The **single choke point** is `governor.py` – it is the only file that creates `ActionRequest` and calls `ExecuteBoundary`. `governor_mediator.py` is purely for input parsing and invocation detection; it does **not** create action requests directly.

```
User Input → WebSocket → governor_mediator.mediate()
    → (if execution requested) → governor.py (SOLE CHOKE POINT)
        → CapabilityRegistry (pre‑action check with absolute path resolution)
        → ExecuteBoundary (phase gate, resource limits)
            → ConfirmationGate (if required; tokens invalidated on session end)
                → Executor (dumb, bounded) → uses NetworkMediator for any I/O
                    → ActionResult → Ledger → UI
```

All outbound network communication passes through **NetworkMediator** (under Governor authority). No other module may import `requests`, `httpx`, `urllib`, `socket`, or similar.

### Key Components

| Component            | Responsibility                                                                 |
|----------------------|---------------------------------------------------------------------------------|
| **governor_mediator.py** | Parse user input, detect explicit invocation, forward to governor.py. **No execution authority. No ActionRequest creation.** |
| **governor.py**      | **Single authority choke point.** Creates ActionRequest, calls CapabilityRegistry, ExecuteBoundary, ConfirmationGate, Executor. |
| **CapabilityRegistry** | Versioned, static registry of all capabilities with unique IDs. Governor consults **before** ActionRequest creation. Uses absolute path resolution. |
| **ExecuteBoundary**  | Enforce phase lock, resource limits (10s timeout, 100MB memory, 5s network, concurrency 1). Fail‑closed. |
| **ConfirmationGate** | Require user confirmation for destructive/irreversible actions. Returns token; tokens invalidated on session end. |
| **ActionRequest**    | Immutable data structure describing the action. Created **only** in governor.py. |
| **ActionResult**     | Standardised result object. |
| **PermissionLedger** | Append‑only log of every action attempt (including denials). |
| **NetworkMediator**  | Single choke point for all external network calls; enforces timeouts, logging, and consent. |

All execution code resides in `executors/` and is reachable **only** via governor.py. Skills may construct ActionRequests but never execute them directly.

### New Safeguard Items (from Action Plan)
- ✅ **Capability Registry** implemented at `src/governor/capability_registry.py` with absolute path resolution. Registry consulted **before** ActionRequest creation. If `registry.json` is missing, malformed, or unreadable, execution fails closed.
- ✅ **Model Version Lock** enforced for all local models. Version hash includes weights, system prompt, wrapper code, and inference parameters. Updates require `MODEL_UPDATED` ledger event + user confirmation. Headless startup blocks inference until confirmed.
- ✅ **External LLM calls** registered as capabilities with `data_exfiltration` flag (design ready; not yet active).
- ✅ **Device Authority Scopes** designed (deferred to Phase 4.5) – see `design/device_authority_scopes.md`.
- ✅ **Network I/O Governor mediation** – all outbound communication routes through `src/governor/network_mediator.py`. CI rule enforces no `requests`, `httpx`, `urllib`, `socket`, `subprocess curl` outside `network_mediator.py`.
- ✅ **Ephemeral session boundary** mechanically enforced: session ID regenerated on reset, timeout, restart; all conversational state cleared; confirmation tokens invalidated.
- ✅ **Influence containment roadmap** documented – `design/influence_containment_roadmap.md`.
- ✅ **Ledger write failure behavior** – if ledger write fails, action execution is denied.

**Unlock Act executed:** A single, reviewed, committed code change in `nova_config.py` set `GOVERNED_ACTIONS_ENABLED = True` (hard‑coded, no env override). This change references the admission gate and evidence bundle.

| Risk | Mitigation | Status |
|------|------------|--------|
| **Bypass of governor.py** | Static analysis ensures only governor.py creates ActionRequests; governor_mediator.py forwards only. | ✅ Verified |
| **Registry path fragility** | Absolute path resolution relative to module; tested with different cwd. | ✅ Verified |
| **Registry unavailability** | Explicit fail‑closed: if registry missing/malformed, no actions proceed. | ✅ Verified |
| **Split authority** | governor.py is sole choke point; all execution flows converge there. | ✅ Verified |
| **Direct network calls outside NetworkMediator** | CI import audit bans `requests`, etc., outside `network_mediator.py`. | ✅ Verified |
| **Model version drift** | Version hash includes weights, prompt, wrapper, parameters; update requires confirmation; headless mode blocks inference. | ✅ Verified |
| **Confirmation token reuse across sessions** | Tokens invalidated on session end; session ID regenerated on reset/timeout/restart. | ✅ Verified |
| **Session persistence** | Session manager explicitly clears all conversational state, corrections, pending tokens. | ✅ Verified |
| **Environment variable override** | `GOVERNED_ACTIONS_ENABLED` hard‑coded in `nova_config.py`; no env reads. | ✅ Verified |
| **Silent fallback** | All fallback paths require user confirmation; code audit ensures none exist. | ✅ Verified |
