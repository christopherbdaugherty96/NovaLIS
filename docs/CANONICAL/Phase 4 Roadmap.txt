## 🔹 Executive Summary

Phase 4 introduces **governed execution** – the ability for Nova to perform explicit, user‑invoked actions with real‑world effects. All execution remains:
- **Explicitly invoked** – no autonomy, no background, no prediction
- **Governor‑mediated** – every action passes through the single choke point
- **Auditable** – every attempt recorded in the immutable ledger
- **Constitutionally constrained** – all Tier 1 invariants remain absolute

This roadmap integrates **four lightweight safeguards** and **adversarial hardening** to ensure real‑world safety without constitutional bloat. It has been reviewed against real code risks and is now **mechanically ready for implementation**.

**The four safeguards are:**
1. **Capability Registry & Lifecycle** – track, version, and revoke every capability.
2. **Model Version Lock** – prevent silent behavioral changes from model/API updates.
3. **External LLM as Governed Capability** – all external model calls are registered, logged, and user‑consented.
4. **Device Authority Scopes** – granular permission levels per domain (designed now, implemented in Phase 4.5).

**Three architectural clarifications are now required:**
- All external network I/O routes through a Governor‑mediated `NetworkMediator`.
- Ephemeral session boundaries mechanically enforced (state cleared on reset/timeout/restart).
- Influence containment roadmap documented before Phase 4.5.

## 🎯 Phase 4 Goals

1. **Enable safe, explicit execution** of a curated set of low‑risk actions.
2. **Prove the Governor Spine** works as the single authority for all execution.
3. **Establish the ConfirmationGate** for actions that require user consent.
4. **Implement the ExecuteBoundary** as the fail‑closed gatekeeper.
5. **Create the ActionRequest / ActionResult** data contracts and enforce them.
6. **Integrate the Permission Ledger** for complete auditability.
7. **Validate all Tier 1 invariants** remain unbroken under execution load.
8. **Build and integrate the four new safeguards** as part of the core implementation.

## 🏛️ Phase 4 Architecture Overview

All execution flows must integrate with the existing Governor components. The **single choke point** is `governor.py` – it is the only file that creates `ActionRequest` and calls `ExecuteBoundary`. `governor_mediator.py` is purely for input parsing and invocation detection; it may **not** create action requests directly.

```
User Input → WebSocket → governor_mediator.mediate()
    → (if execution requested) → governor.py (SOLE CHOKE POINT)
        → CapabilityRegistry (pre‑action check with absolute path resolution)
        → ExecuteBoundary (phase gate, resource limits)
            → ConfirmationGate (if required; tokens invalidated on session end)
                → Executor (dumb, bounded) → uses NetworkMediator for any I/O
                    → ActionResult → Ledger → UI
```

All outbound network communication must pass through **NetworkMediator** (under Governor authority). No other module may import `requests`, `httpx`, `urllib`, `socket`, or similar.

### Key Components

| Component            | Responsibility                                                                 |
|----------------------|---------------------------------------------------------------------------------|
| **governor_mediator.py** | Parse user input, detect explicit invocation, forward to governor.py. **No execution authority. No ActionRequest creation.** |
| **governor.py**      | **Single authority choke point.** Creates ActionRequest, calls CapabilityRegistry, ExecuteBoundary, ConfirmationGate, Executor. |
| **CapabilityRegistry** | Versioned, static registry of all capabilities with unique IDs. Governor consults **before** ActionRequest creation. Uses absolute path resolution. |
| **ExecuteBoundary**  | Enforce phase lock, resource limits (10s timeout, 100MB memory, 5s network, concurrency 1). Fail‑closed. |
| **ConfirmationGate** | Require user confirmation for destructive/irreversible actions. Returns token; tokens invalidated on session end. |
| **ActionRequest**    | Immutable data structure describing the action. Created **only** in governor.py. |
| **ActionResult**     | Standardised result object. |
| **PermissionLedger** | Append‑only log of every action attempt (including denials). |
| **NetworkMediator**  | Single choke point for all external network calls; enforces timeouts, logging, and consent. |

All execution code resides in `actions/` and is reachable **only** via governor.py. Skills may construct ActionRequests but never execute them directly.

## 🔒 Phase 4 Admission Gate (Pre‑Unlock Checklist)

Before `GOVERNED_ACTIONS_ENABLED = True` can be committed, the following must be **mechanically verified**. Items marked **[Adversarial]** are direct responses to identified risk zones.

### Original Items (from Phase 4 Roadmap)
- [x] Phase 3.5 sealed and frozen.
- [ ] **Single Master Governor proof** – No code path bypasses governor.py. Verified by static analysis and adversarial tests. **[Adversarial]** Confirm that governor_mediator.py does **not** create ActionRequests.
- [ ] **CI enforces execution gating** – In all runtime contexts, `GOVERNED_ACTIONS_ENABLED = False` in `nova_config.py`. Environment variable override is impossible.
- [ ] **Governor Spine** fully implemented and tested (explicit invocation, one‑strike clarification, verbatim echo, offline‑first).
- [ ] **All Phase 4 capabilities** registered in `registry.json` with unique IDs (as per schema).
- [ ] **ExecuteBoundary** implemented with fail‑closed default and resource guardrails.
- [ ] **ConfirmationGate** implemented for required action types; confirmation tokens expire on session end.
- [ ] **ActionRequest/ActionResult** data structures finalised and used throughout; ActionRequest creation **only** in governor.py.
- [ ] **Permission Ledger** operational, append‑only, with pruning policy defined and logged.
- [ ] **Adversarial test suite** passes – includes attempts to bypass Governor, execute without invocation, exceed limits, etc.
- [ ] **Dynamic execution surface ban** verified – no `exec`, `eval`, `subprocess` (except STT pipeline) in authority paths.
- [ ] **Environment variable override lock** – `GOVERNED_ACTIONS_ENABLED` is hard‑coded; any attempt to read from env is absent.
- [ ] **Zero silent fallback patterns** – any fallback must be user‑confirmed.
- [ ] **Multi‑user boundary** – Phase 4 remains single‑user; any multi‑user logic is absent.
- [ ] **Ledger purge governance** – manual purge requires Governor mediation and logs `CONSTITUTIONAL_PRUNE` event.
- [ ] **All tests** pass in a clean environment.

### New Items (from Safeguards & Adversarial Review)
- [ ] **Capability Registry** implemented at `src/governor/capability_registry.py` with **absolute path resolution** (relative to module, not cwd). Registry consulted **before** ActionRequest creation. **If `registry.json` is missing, malformed, or unreadable, execution fails closed.**
- [ ] **Model Version Lock** enforced for all local and external models. Version hash includes:
  - weights hash
  - system prompt hash
  - wrapper logic hash
  - default parameters (temperature, top_p, max_tokens, etc.)
  Updates require `MODEL_UPDATED` ledger event + user confirmation. **[Adversarial]** If headless startup, block inference until confirmation; no auto‑approval.
- [ ] **External LLM calls** registered as capabilities with `data_exfiltration` flag; user consent enforced via NetworkMediator.
- [ ] **Device Authority Scopes** designed (deferred to Phase 4.5) – see `design/device_authority_scopes.md`.
- [ ] **Network I/O Governor mediation** – all outbound communication routes through `src/governor/network_mediator.py`. **[Adversarial]** CI rule: fail build if `import requests`, `httpx`, `urllib`, `socket`, `subprocess curl` appears outside `network_mediator.py` (and allowed STT pipeline).
- [ ] **Ephemeral session boundary** mechanically enforced:
  - Session ID regenerated on reset, timeout (30 min), or restart.
  - All conversational state cleared (conversation buffer, quick_corrections staging, pending confirmation tokens).
  - Confirmation tokens invalidated on session end.
- [ ] **Influence containment roadmap** documented – `design/influence_containment_roadmap.md`.
- [ ] **Ledger write failure behavior** – If ledger write fails, action execution is denied.

**Unlock Act:** A single, reviewed, committed code change in `nova_config.py`:
```python
GOVERNED_ACTIONS_ENABLED = True   # immutable, no env override
```
Must reference this admission gate and include evidence of all checklist items.

## ⚠️ Risk Zones & Mitigations (Adversarially Verified)

| Risk | Mitigation |
|------|------------|
| **Bypass of governor.py** | Static analysis ensures only governor.py creates ActionRequests; governor_mediator.py forwards only. |
| **Registry path fragility** | Absolute path resolution relative to module; test with different cwd. |
| **Registry unavailability** | Explicit fail‑closed: if registry missing/malformed, no actions proceed. |
| **Split authority** | governor.py is sole choke point; all execution flows converge there. |
| **Direct network calls outside NetworkMediator** | CI import audit bans `requests`, etc., outside `network_mediator.py`. |
| **Model version drift** | Version hash includes weights, prompt, wrapper, parameters; update requires confirmation; headless mode blocks inference. |
| **Confirmation token reuse across sessions** | Tokens invalidated on session end; session ID regenerated on reset/timeout/restart. |
| **Session persistence** | Session manager explicitly clears all conversational state, corrections, pending tokens. |
| **Environment variable override** | `GOVERNED_ACTIONS_ENABLED` is hard‑coded in `nova_config.py`; no env reads. |
| **Silent fallback** | All fallback paths require user confirmation; code audit ensures none exist. |
| **Ledger write failure** | If ledger write fails, action execution is denied; no action proceeds unlogged. |

## ✅ Success Criteria for Phase 4 Unlock

- All admission gate checklist items completed and verified.
- No constitutional violations observed during extended testing (30+ days simulated usage).
- All Phase 4 capabilities functional and correctly mediated.
- Governor bypass proof updated and confirmed.
- Ledger operational and tamper‑evident; ledger write failure blocks execution.
- ConfirmationGate behaves as specified; tokens expire on session end.
- ExecuteBoundary correctly denies actions when phase is locked or limits exceeded.
- Capability Registry enforces enable/disable and unique IDs; path resolution robust (tested from multiple cwds); missing registry fails closed.
- Model Version Lock blocks unapproved model updates; headless mode safe.
- NetworkMediator logs all external calls and respects rate limits; import audit passes.
- Session boundary clearing verified; no cross‑session state leakage.
