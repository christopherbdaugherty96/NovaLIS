## 🔴 Critical Issues

### C-1 — Hardcoded Absolute Windows Path in `stt_engine.py`
```python name=nova_backend/src/services/stt_engine.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/services/stt_engine.py#L15
FFMPEG_PATH = r"C:\Nova-Project\nova_backend\tools\ffmpeg\ffmpeg.exe"

**Problem:** This path is machine-specific. The server will crash silently on any other machine or in CI/CD. There is no fallback to `ffmpeg` on `PATH`.

**Fix:**
```python name=stt_engine_fix.py
import shutil
FFMPEG_PATH = shutil.which("ffmpeg") or r"C:\Nova-Project\nova_backend\tools\ffmpeg\ffmpeg.exe"
if not FFMPEG_PATH:
    raise RuntimeError("ffmpeg not found. Install it or place it at the expected path.")
```

### C-2 — Leaked Temporary File in `stt_manager.py`
```python name=nova_backend/src/stt_manager.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/stt_manager.py#L71-L80
def transcribe_audio_bytes(audio_bytes: bytes) -> str:
    with tempfile.NamedTemporaryFile(suffix=".wav", delete=False) as tmp:
        tmp.write(audio_bytes)
        tmp_path = tmp.name
    # ... tmp_path is NEVER CLEANED UP
```
**Problem:** `delete=False` + no cleanup = temp files accumulate on disk indefinitely. High-use systems will fill the disk.

**Fix:** Use `try/finally` or a `TemporaryDirectory` (as correctly done in `stt_engine.py`):
```python name=stt_manager_fix.py
import os
import tempfile

def transcribe_audio_bytes(audio_bytes: bytes) -> str:
    tmp_path = None
    try:
        with tempfile.NamedTemporaryFile(suffix=".wav", delete=False) as tmp:
            tmp.write(audio_bytes)
            tmp_path = tmp.name
        # ... transcription logic ...
    finally:
        if tmp_path and os.path.exists(tmp_path):
            os.unlink(tmp_path)
```

### C-3 — Dual STT Systems Creating Confusion
There are **two separate STT implementations** that are not cleanly separated:
- `nova_backend/src/services/stt_engine.py` — **Vosk-based**, used by the live router
- `nova_backend/src/stt_manager.py` — **Whisper+OpenAI hybrid**, appears to be legacy/unused

**Problem:** `stt_manager.py` imports `faster_whisper` and `openai` at module level (not inside the router), meaning if either package is present, they are loaded at startup — potentially including the OpenAI client initialization with an API key. This increases startup time and creates a confusing dual-path. It's unclear which is canonical.

**Recommendation:** If `stt_manager.py` is legacy, move it to `archive_quarantine/`. If it's intended for future use, gate its imports inside functions and add a clear `# FUTURE USE - NOT ACTIVE` header.

## 🟠 Security Issues

### S-1 — `OPENAI_API_KEY` Loaded Unconditionally at Module Import
```python name=nova_backend/src/stt_manager.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/stt_manager.py#L16-L18
OPENAI_KEY = os.getenv("OPENAI_API_KEY")
client = OpenAI(api_key=OPENAI_KEY) if OPENAI_KEY else None
```
**Problem:** If this file is ever imported (directly or transitively), it silently reads the API key from the environment and initializes a client. Even if the function is never called, the key is in memory. This is a secrets hygiene concern.

**Fix:** Lazy-initialize inside the function only when needed:
```python name=stt_manager_lazy_fix.py
def _get_openai_client():
    key = os.getenv("OPENAI_API_KEY")
    if not key:
        return None
    from openai import OpenAI
    return OpenAI(api_key=key)
```

### S-2 — WebSocket Input Not Validated Against Maximum Length
```python name=nova_backend/src/brain_server.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/brain_server.py#L128-L135
raw = await ws.receive_text()
msg = json.loads(raw)
text = (msg.get("text") or "").strip()
```
**Problem:** There's no maximum length check on the incoming WebSocket message. A malicious or buggy client could send arbitrarily large payloads, causing memory pressure or slow processing by the LLM skill.

**Fix:**
```python name=ws_length_guard.py
MAX_INPUT_BYTES = 4096
raw = await ws.receive_text()
if len(raw) > MAX_INPUT_BYTES:
    await send_chat_message(ws, "Input too long.")
    await send_chat_done(ws)
    continue
```

### S-3 — No CORS Policy Defined
The FastAPI app has no CORS middleware configured. Since the frontend and backend share the same origin in deployment (via static file mounting), this is acceptable — but if the dashboard is ever served from a different origin (e.g., `Nova-Frontend-Dashboard/` served separately during development), cross-origin requests would either fail silently or be allowed by browser default.

**Fix (Explicitly Lock It Down):**
```python name=cors_fix.py
from fastapi.middleware.cors import CORSMiddleware
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://127.0.0.1:8000"],  # Same-origin only
    allow_methods=["GET", "POST"],
    allow_headers=["*"],
)
```

### B-1 — `SpeechState` Missing `stop()` Method Used in `brain_server.py`
```python name=nova_backend/src/brain_server.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/brain_server.py#L143-L146
if lowered == "stop":
    speech_state.stop()   # ← AttributeError at runtime!
```
```python name=nova_backend/src/speech_state.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/speech_state.py#L1-L35
class SpeechState:
    def __init__(self) -> None:
        self.last_spoken_text: Optional[str] = None
    # ← No stop() method defined!
```
**Problem:** `brain_server.py` calls `speech_state.stop()` but `SpeechState` has no `stop()` method. This will cause an `AttributeError` at runtime the first time a user says "stop".

**Fix:**
```python name=speech_state_fix.py
class SpeechState:
    def __init__(self) -> None:
        self.last_spoken_text: Optional[str] = None

    def stop(self) -> None:
        """Signal stop; clears last spoken text."""
        self.last_spoken_text = None
```

### B-2 — `AudioManager` Creates asyncio Task in `__init__` at Import Time
```python name=nova_backend/src/audio_manager.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/audio_manager.py#L40-L49
try:
    loop = asyncio.get_running_loop()
except RuntimeError:
    loop = asyncio.new_event_loop()
    asyncio.set_event_loop(loop)

loop.create_task(self._worker())
```
**Problem:** This is called at module import time (`audio_manager = AudioManager()`). If no event loop is running at import time, a new one is created and set as current — this can conflict with FastAPI/uvicorn's own event loop, causing subtle async bugs or the worker task never actually running.

**Fix:** Use `asyncio.ensure_future()` lazily, or start the worker via a FastAPI `lifespan` startup event:
```python name=audio_manager_lifespan_fix.py
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app):
    asyncio.create_task(audio_manager._worker())
    yield

app = FastAPI(lifespan=lifespan)
```

### B-4 — `GeneralChatSkill` Uses Synchronous `ollama.chat()` in an `async` Function
```python name=nova_backend/src/skills/general_chat.py url=https://github.com/christopherbdaugherty96/NovaLIS/blob/e63b47d12f363fc9a9a99a29b66a6926dde35cc9/nova_backend/src/skills/general_chat.py#L201-L220
async def handle(self, query: str) -> SkillResult | None:
    ...
    response = ollama.chat(...)   # ← Synchronous blocking call in async context!
```
**Problem:** `ollama.chat()` is a blocking network call. Calling it directly in an `async` function **blocks the entire asyncio event loop**, freezing all WebSocket connections during inference. This is a classic async antipattern.

**Fix:** Run it in a thread pool:
```python name=general_chat_async_fix.py
import asyncio
loop = asyncio.get_event_loop()
response = await loop.run_in_executor(None, lambda: ollama.chat(
    model="phi3:mini",
    messages=[...],
    options={...}
))
```
