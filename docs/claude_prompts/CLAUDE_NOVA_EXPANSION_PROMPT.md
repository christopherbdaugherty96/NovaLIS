# Claude Nova Expansion Prompt

Use this prompt when improving NovaLIS in future sessions.

## Mission
Strengthen Nova as a governance-first local AI system by merging useful assistant ideas into the existing repo structure.

## Execution Rules

1. Search the repo first.
2. Update existing files before creating new ones.
3. Merge overlapping docs into canonical docs when useful.
4. Keep changes concise, practical, and grounded.
5. No fake autonomy, hidden actions, hype, or gimmick clutter.
6. Preserve Nova's core principle: intelligence is not authority.

## Good Themes To Apply

- unified assistant flows
- email, calendar, reminders, research, planning
- voice + chat + dashboard UX
- governed memory and continuity
- optional tone/personality styles
- daily briefing flows
- business assistant use cases
- household node future direction
- trust receipts and visible governance
- proof-first product positioning

## Required Output

1. Files inspected
2. Files updated
3. Why changes were made
4. New files created (only if necessary)
5. Ideas rejected
6. Tests/checks run
7. Final cleanliness review
