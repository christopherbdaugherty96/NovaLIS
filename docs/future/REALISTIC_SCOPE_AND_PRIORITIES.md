# Realistic Scope and Priorities

Status: planning guardrail / future scope control

This document exists to keep future-direction ideas honest, focused, and executable.

It distinguishes what is realistic now, what is realistic later, and what should not be treated as near-term scope.

It also recognizes that Auralis Digital already exists as a separate completed project/business asset. Nova should complement it, not duplicate it.

---

## Core Rule

Broad vision is useful.
Unbounded scope is dangerous.

The main risk is not lack of ideas. The main risk is trying to build too many businesses and products inside one roadmap at the same time.

---

## Existing Asset: Auralis Digital

Auralis Digital is an existing separate project/business asset.

Practical implication:

- do not rebuild Auralis Digital inside Nova
- use it as a market-facing revenue vehicle if desired
- let Nova become internal leverage where useful
- keep brand identities clear

Nova and Auralis Digital can complement each other without becoming the same thing.

---

## Scope Ratings

## Tier 1 — Realistic Now

These can create value immediately with focused execution.

### A. Website Services via Auralis Digital

Reality: very realistic now.

Use existing tools and manual delivery.
Nova can assist internally with drafts, copy, workflows, and planning.

Near-term scope:

- client intake workflows
- copy drafting
- site structure suggestions
- revision support
- proposal generation
- maintenance workflows

Do not require Nova runtime breakthroughs.

### B. Internal Workflow Assistant

Reality: very realistic now.

Use Nova principles for your own operations.

Near-term scope:

- checklists
- proposals
- templates
- research summaries
- campaign planning
- task organization

### C. Draft-Only Content Studio

Reality: realistic now if tightly scoped.

Near-term scope:

- hooks
n- scripts
- captions
- thumbnail ideas
- content calendars
- export packages

No autonomous posting required.

---

## Tier 2 — Realistic Later (After Focused Foundations)

### D. Shopify Product Marketing Engine

Reality: realistic later with bounded scope.

Good first version:

- product in
- campaign package out

Needed foundations:

- clean product data
- profile system
- review UI
- claim checks

### E. Multi-Profile Growth System

Reality: possible but more complex than it appears.

Needed foundations:

- state management
- separate memory contexts
- analytics model
- UI controls
- quality safeguards

### F. Semi-Automated Website Production Engine

Reality: realistic as a draft accelerator.

Honest scope:

- first drafts
- layouts
- copy
- reusable sections
- style systems

Human finishing still required.

---

## Tier 3 — High Risk / Not Near-Term Scope

### G. Fully Autonomous Agency

Not near-term realistic.

Reason:
real clients, revisions, edge cases, support, billing, and communication are messy.

### H. Passive Income Auto Channel Machine

Overhyped if treated as guaranteed outcome.

Useful as an assistant workflow, unreliable as a promise.

### I. Everything Platform

Nova should not try to become every product category at once.

---

## Recommended Priority Order

1. Use Auralis Digital for real revenue now.
2. Build one Nova MVP feature only.
3. Reinvest learning into next capability.
4. Expand only after one workflow works reliably.

---

## What To Build First Inside Nova

Choose one:

- Trust Review UI
n- Content Studio
- Shopify campaign generator
- Profile system
- Internal operations assistant

One clear win is more valuable than five partial systems.

---

## Non-Goals Right Now

Avoid near-term distraction from:

- autonomous ad spend
- auto-replying everywhere
- broad connector sprawl
- replacing full agencies overnight
- chasing every niche simultaneously

---

## Honest Final Framing

Nova is strongest as a governed foundation.
Auralis Digital is strongest as a direct business asset.

Use the business to create revenue.
Use the product to create long-term leverage.
