Nova is a governed household intelligence presence вҖ” a calm, trusted interface for all digital life within the home.

Nova is not a chatbot, assistant, agent, or product designed to capture attention.

Nova is a locally-hosted, constitutionally-governed household intelligence system that gives the user safe access to powerful AI capabilities without surrendering authority.

Core invariant (immutable, all phases):

Intelligence may expand. Authority remains governed.

Permanent execution path (all phases):

User вҶ’ Governor вҶ’ Capability вҶ’ Ledger вҶ’ Result
Nothing executes without:

Explicit user invocation OR active user-defined policy
Governor validation against law
Phase authorization

Layer 1 вҖ” Law:          User-defined policies. Immutable at runtime. Changed only by POLICY_REPLACED events.

4.2 WebSocket Interaction Flow
Client connects to /ws
  вҶ’ Server sends greeting:
      {"type":"chat","message":"Hello. How can I help?"}

  вҶ’ Client sends: {"text": "user input here"}

      1. Fast-path commands (stop вҶ’ speech_state.stop(), repeat вҶ’ resend last)
      2. GovernorMediator.parse_governed_invocation() вҶ’ Invocation | Clarification | None
         - "search [query]" / "look up [query]" вҶ’ Invocation(capability_id=16, params={query})
         - "open [name]" вҶ’ Invocation(capability_id=17, params={target})

      3. If governed invocation вҶ’ Governor.handle_governed_invocation()
      4. Else вҶ’ SkillRegistry.handle_query() (deterministic, single-pass)

  вҶ’ Server sends widget or chat response + {"type":"chat_done"}

PART 5 вҖ” GOVERNOR SPINE DETAIL (Phase 4 Full Spec)
5.1 Governor (governor/governor.py)
Single authority choke point вҖ” owns ExecuteBoundary + SingleActionQueue
handle_governed_invocation(capability_id, params) is the only path to execution
Validates: capability exists вҶ’ enabled вҶ’ ExecuteBoundary.allow_execution() вҶ’ Ledger writable вҶ’ executor runs
5.2 GovernorMediator (governor/governor_mediator.py)
Phase-4 parser shim вҖ” deterministic regex detection
SEARCH_RE: matches search [for], look up, research + query
OPEN_RE: matches open [name]
Ephemeral one-strike clarification state per session_id вҖ” must call clear_session() on disconnect
Returns Invocation | Clarification | None
Never creates ActionRequests. Never executes anything.
5.3 ExecuteBoundary (governor/execute_boundary/execute_boundary.py)
GOVERNED_ACTIONS_ENABLED = True (Phase 4 staging)
allow_execution() returns this flag
MAX_CONCURRENT = 1 enforced via SingleActionQueue
MAX_EXECUTION_TIME = 10 вҖ” defined but NOT enforced (placeholder вҖ” BUG-Q10)
5.4 CapabilityRegistry (governor/capability_registry.py)
Loads config/registry.json at init вҖ” fail-closed if missing or malformed
Validates: schema_version = "1.0", phase = "4", required fields, risk_level, status, enabled
is_enabled() = status == "active" AND enabled == True
Planned extension: authority_class (A/B/C/D/E) and timeout_ms fields (Part 20)
5.5 NetworkMediator (governor/network_mediator.py)
Sole gateway for all external HTTP(S) calls
Enforces: 5s timeout, SSRF protection, rate limiting (50 req/min, thread-safe), Ledger logging
Synchronous (requests) вҖ” async callers must use asyncio.to_thread()
5.6 LedgerWriter (ledger/writer.py)
Logs: ACTION_ATTEMPTED, ACTION_COMPLETED, ACTION_FAILED, POLICY_REPLACED
Must be writable before any execution effect вҖ” fail-closed on LedgerWriteFailed
Append-only. Immutable once written.
Gap: No structured query engine вҖ” see Part 20, Gap #2
5.7 ActionRequest (actions/action_request.py)
Immutable frozen dataclass with MappingProxyType params
Created only in governor.py
Fields: capability_id, params, request_id (UUID), created_at (ISO-8601 UTC)

## PART 11 вҖ” PHASE 6 DESIGN SPEC (Policy-Bound Delegated Autonomy)
**Status:** Design-Unlocked / Runtime-Locked until Phase 5 complete + 3 months deployment**Date:** 2026-02-16
### 11.1 The Shift
> Governor question changes from *"Did the user explicitly request this?"* to *"Is this action permitted by the active delegation policy?"*
This is a **policy-validation model**, not a trust model.
### 11.2 What Phase 6 Allows
| Feature | Description | Hard Constraint |
|---|---|---|
| **Atomic Policies** | User defines: trigger + one atomic action | Single capability from Phase 4 registry вҖ” no multi-step chains |
| **Policy Envelope** | Scope, resource limits, frequency caps | May not compose new capabilities |
| **Background Monitoring** | Trigger-only evaluation | No speculative reasoning; external OS timers only; all monitoring logged |
| **Deterministic Evaluation** | Multiple triggered policies вҶ’ alphabetical by policy ID | No dynamic prioritisation |
| **Class A Mutations** | Adjust retrieval ranking weights (Tier 3 + Evolution ON) | Reversible, delta-limited, logged |
| **Class B Mutations** | Memory compaction and pruning | Before-state snapshot; user can roll back |
| **Class D Mutations** | Non-authority skill routing hints | No effect on capability registry |
| **Emergency Stop** | Unregisters ALL external timers and listeners | Must execute within 100ms |
### 11.3 Phase 6 Architecture
User-defined Atomic Policies (Ring 3) вҶ“ Governor Policy Validator вҶ“ Background Monitor (Trigger-Only вҖ” external OS timers, no Nova loops) вҶ“ Governor: policy enabled? within envelope? deterministic order? вҶ“ ExecuteBoundary вҶ’ Atomic Action вҶ’ Ledger

### 11.4 Mutation Framework Conditions
1. Evolution Kill Switch = ON
2. Autonomy Tier вүҘ 3
3. Target component is `@mutable` and in Ring 2
4. Delta limits and frequency caps satisfied
5. After-state simulation matches claimed state (`component.simulate(delta)`)

**Mutations may NEVER modify:** policy definitions, policy envelopes, capability registry.
### 11.5 Firewall & Ledger Specification (Adversarial-Grade)
- **8-stage validation pipeline** per mutation
- **Exclusive mutation lock** per component (pre-snapshot through commit)
- **After-state revalidation** вҖ” mismatch = rejection
- **Zero alternative write paths** вҖ” build-time invariant
- **Ring 1 Python protection** вҖ” frozen dataclasses + `__setattr__` veto + closure encapsulation
- **Extended Ledger events:** `POLICY_CREATED`, `POLICY_ENABLED`, `POLICY_DISABLED`, `AUTONOMOUS_ACTION_ATTEMPT`, `MUTATION_APPLIED`, `MUTATION_REJECTED`, `MUTATION_ROLLED_BACK`
