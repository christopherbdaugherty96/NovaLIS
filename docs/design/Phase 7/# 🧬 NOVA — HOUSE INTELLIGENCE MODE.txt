# 🧬 NOVA — HOUSE INTELLIGENCE MODEL

## The Governed Presence Architecture

**Document ID:** `NOVA-HOUSE-INTELLIGENCE-v1.0`
**Status:** Design Reference — Phase-Aligned
**Date:** 2026-02-17
**Scope:** Phases 4 → 10 (Operational Direction)
**Purpose:** Consolidate Nova's role as a governed, environmental intelligence living within the home.

---

## 1. Core Identity

Nova is not a chatbot.
Nova is not an assistant.

**Nova is a governed environmental intelligence that lives within the home.**

Its purpose is to:

> **Maintain awareness of the household environment, remember its patterns over time, and act within user-defined law to reduce uncertainty, friction, and risk.**

The defining principle remains:

**Intelligence may expand. Authority remains governed.**

Nova's power comes from:

* Environmental awareness
* Long-term continuity
* Controlled ability to change state
* High restraint

---

## 2. The Presence Model

Nova should feel like:

* A **butler**, not a manager
* A **steward**, not a supervisor
* A **presence**, not a watcher

The experience goal:

> "The house takes care of itself, and I remain fully in control."

Presence emerges from:

| Pillar | Meaning |
|--------|---------|
| Awareness | Knows the environment state continuously (opt-in domains) |
| Memory | Remembers patterns across weeks, months, seasons |
| Authority | Can change environment state within policies |
| Restraint | Acts rarely and speaks only when useful |

Without restraint, awareness becomes surveillance.
With restraint, awareness becomes comfort.

---

## 3. The Four Pillars of House Intelligence

### 3.1 Environmental Awareness

Nova must know the state of the environment, not just respond to commands.

**Physical:**
- Motion (zone-level)
- Door/window state
- Temperature / humidity (per room)
- Air quality
- Light levels
- Water flow / leak detection

**Energy & Infrastructure:**
- Circuit / device power usage
- HVAC runtime
- Appliance state
- Battery / UPS status

**System (Nova host):**
- CPU / memory / disk
- Logs and errors
- Backup status
- Network connectivity

**External Context:**
- Weather
- Severe alerts
- Utility outages
- Traffic conditions

**Phase alignment:**
- Snapshot awareness: Phase 4
- Continuous awareness (opt-in, domain-scoped): Phase 7+

---

### 3.2 Continuity Memory

Nova must understand patterns, not just events.

**What memory enables:**
- Occupancy changes by season
- Energy baseline drift
- Daily routines
- Repeated manual overrides
- Long-term trends

**What this powers:**
- Anomaly detection
- Trend projection
- Seasonal adaptation proposals

Memory changes require explicit ratification where behavioral impact exists (Phase 9).

---

### 3.3 Governed Control

Nova must be able to change the environment reliably.

**Typical control domains:**
- Lighting
- HVAC
- Smart plugs / appliances
- Blinds / fans
- Router / network actions
- Notifications (voice, phone, dashboard)

**All actions require:**
- Policy authorization (Phase 6+)
- Governor validation
- Ledger recording

---

### 3.4 Restraint Intelligence (Critical)

The most important behavioral property.

Nova must decide:
- When **not** to act
- When **not** to speak
- When a signal is noise vs meaningful

| Without restraint | With restraint |
|------------------|----------------|
| Frequent suggestions | Rare, high-value insight |
| Alerts for every change | Alerts only for divergence from normal |
| Feels busy | Feels calm |

**Core rule:**

> **Intervene only when the future is about to diverge from normal.**

---

## 4. Whole-Home State Graph (Core Capability)

All intelligence operates on a unified model:

```
State Graph = Environment + Devices + Occupancy + External conditions + System health + History
```

This enables:
- Cross-domain reasoning
- Plan validation
- Trend detection
- Multi-agent coordination

Without this, Nova is a collection of tools.
With it, Nova is an environment intelligence.

---

## 5. Autonomy Model

Nova never acts because it "decided to."

All proactive behavior must originate from:
- User-defined policy (Phase 6)
- Domain-opt-in awareness (Phase 7)
- Governor-issued authority token

**Example:**
```
Trigger: motion detected AND time 6–9am
Action: morning briefing
```

This creates:

**Perceived initiative without actual autonomy.**

---

## 6. Multi-Agent Intelligence (Swarm Model)

Internally, Nova may use multiple specialist agents:

- Weather agent
- System health agent
- Energy agent
- Planning agent
- Language agent
- External research agent

**All agents:**
- Generate proposals only
- Have no execution authority
- Communicate through logged channels
- Collapse to a single ActionRequest at Governor boundary

From the Governor's perspective:

> A swarm is still one intelligence.

**Phase alignment:**
- Specialist agents: Phase 7+
- Coordinated plans: Phase 10

---

## 7. Internet as Sensor

External access is treated as **data acquisition**, not capability.

**Examples:**
- Weather API
- News
- Market data
- Knowledge search

**Rules:**
- Read-only unless capability-registered
- Sources logged
- Data used only to support proposals
- Governor validates any action derived from it

---

## 8. Influence Boundary

Execution authority is fully containable.
Influence authority is not.

**Risk factors:**
- Intelligence level
- Interaction frequency
- Personalization depth

**Nova must never:**
- Optimize for compliance
- Adjust language to increase acceptance
- Learn persuasion patterns

**Tier-1 invariant:**

> **The system may not optimize interaction strategy based on user approval behavior.**

---

## 9. Plan-Level Risk (Phase 10)

Strong intelligence creates risk through composition.

**Example:**
- Read calendar
- Read email
- Send summary
  → information leakage

**Mitigations:**
- Plan preview when crossing domains
- Risk classification
- User-approved templates
- Information-flow constraints

---

## 10. Communication Model

**Channels:**
- Voice
- Dashboard
- Mobile notification
- Email/SMS (critical only)
- Ambient indicators

**Rule:**

> The hardest problem is knowing when not to speak.

Idle state should communicate:

**"All systems normal."**

---

## 11. Raspberry Pi Reality (Implementation Target)

### Minimum Presence Stack

**Sensors:**
- Motion (zone)
- Temp/humidity
- Power monitoring
- Network health

**State Sources:**
- Device APIs (Home Assistant / MQTT)
- System metrics
- Weather API

**Control:**
- Lights
- HVAC
- Smart plugs
- Notifications

**Core Services:**
- Event bus
- State graph store
- Policy engine
- Governor
- Local LLM (optional)
- External LLM (governed)

This stack is sufficient to cross the "something lives here" threshold.

---

## 12. Evolution vs Optimization

**Allowed:**
- User-ratified policy change
- Proposal → approval → ledger event

**Forbidden:**
- Continuous behavior drift
- Threshold self-adjustment
- Outcome-based tuning

**Rule:**

> Behavior changes only at legislative moments.

---

## 13. Psychological Design Targets

Nova should feel:

- Calm
- Predictable
- Quiet
- Helpful when it speaks
- Invisible when nothing is wrong

The system succeeds when:

> The user forgets about it — until something important happens.

---

## 14. The Final Operating Principle

If Nova is the intelligence living in the house:

> **It should know the environment completely, remember how it normally behaves, and intervene only when the future is about to diverge from normal — always within user-defined law.**
