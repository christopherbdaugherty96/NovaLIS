Nova: Personal Intelligence Hub Architecture

рҹҸ  EXECUTIVE VISION

Nova transforms from a governed intelligence container into a Personal Intelligence HubвҖ”a local-first, user-sovereign system that orchestrates smart devices, sensors, and data without ever assuming decision authority. It remains fully governed, transparent, and orthogonally structured while becoming the central nervous system for personal environments.

---

рҹҺҜ CORE PRINCIPLES (IMMUTABLE)

1. Local-First Sovereignty

В· Runs on Raspberry Pi, personal server, or dedicated hardware
В· Zero external AI dependencies for core cognition
В· All data stays local unless explicitly exported
В· User owns the entire stack

2. Intelligence-Authority Split Maintained

```
Devices = Data sources & actuators
Nova Agents = Intelligence lenses
User = Decision authority
Governor = Single choke point enforcement
```

3. Orthogonal Integration

Each device/sensor/API becomes an input agent or output agent, not a cognitive agent. They feed data to cognitive agents or execute user-approved actions.

Actuator Output Agents

```python
class ActuatorAgent(Agent):
    """
    Executes device actions with explicit user approval.
    """
    def execute(self, action: Action) -> ExecutionResult:
        # Must have Governor approval
        if not self.governor.approve_action(action):
            return ExecutionResult(
                success=False,
                reason="Governor rejected",
                # NO retry suggestion
            )
        
        # Execute with timeout
        result = self.device.execute(action)
        
        return ExecutionResult(
            success=result.success,
            execution_id=result.id,
            timestamp=now(),
            audit_trail=result.audit_trail
        )
```

Routine Agent

```python
class RoutineAgent(Agent):
    """
    Manages user-defined routines as agent sequences.
    """
    def execute_routine(self, routine_id: str) -> RoutineExecution:
        """
        Runs predefined agent sequence with transparency.
        """
        routine = self.routine_store.get(routine_id)
        
        # Present routine plan to user
        plan = self._create_execution_plan(routine)
        
        # Personality Agent presents plan
        self.personality_agent.present_plan(plan)
        
        # Wait for explicit user confirmation
        if not self._get_user_confirmation():
            return RoutineExecution(
                status="CANCELLED_BY_USER",
                # NO persuasion to continue
            )
        
        # Execute each step with Governor oversight
        results = []
        for step in routine.steps:
            step_result = self.governor.execute_step(step)
            results.append(step_result)
            
            # Present each result as it happens
            self.personality_agent.present_step_result(step, step_result)
            
            # Check for required confirmations
            if step.requires_confirmation:
                if not self._get_user_confirmation():
                    return RoutineExecution(
                        status="PAUSED_AT_STEP",
                        completed_steps=results,
                        # NO "you should continue"
                    )
        
        return RoutineExecution(
            status="COMPLETED",
            results=results,
            audit_trail=self._generate_audit_trail(routine, results)
        )
```

```python
class MultiModalContradictionAgent(Agent):
    """
    Detects contradictions across device/sensor inputs.
    """
    def analyze(self, device_readings: Dict[str, RawData]) -> ContradictionReport:
        report = ContradictionReport()

        indoor_temp = device_readings["living_room_thermostat"]["temperature"]
```

рҹ”җ GOVERNANCE EXTENSIONS FOR DEVICE CONTROL

New Constitutional Amendments

```
Article 11: Device and Routine Governance

11.1 Device Access Principle
     a) Devices are data sources or actuators, not cognitive agents
     b) Device agents output raw data only, no interpretation
     c) Actuator agents require explicit user approval per action
     d) No device may autonomously control another device

11.2 Routine Execution Principle
     a) Routines are predefined sequences of agent invocations
     b) Each routine step's output is presented raw to user
     c) User may pause, modify, or cancel at any step
     d) No routine may include hidden or inferred steps

11.3 Multi-Modal Data Principle
     a) Sensor data is processed locally by default
     b) Privacy filters must be applied before storage/display
     c) Contradictions between sensors are surfaced, not resolved
     d) No sensor fusion that creates synthetic interpretations

11.4 Local-First Principle
     a) Core cognition operates without external AI
     b) External APIs are optional enhancements, not requirements
     c) User data never leaves local environment without explicit export
     d) System remains functional during network outages
```

```python
class DevicePersonalityExtension:
    """
    Extends Personality Agent for device/routine interactions.
    """
    def present_device_data(self, device_readings: Dict) -> str:
        """
        Presents device data without interpretation.
        """
        presentation = []
        
        for device_id, readings in device_readings.items():
            presentation.append(f"### {device_id}")
            presentation.append(f"*Last updated: {readings['timestamp']}*")
            presentation.append("")
            
            # Present raw readings
            for key, value in readings['data'].items():
                presentation.append(f"- {key}: {value} {readings.get('units', {}).get(key, '')}")
            
            presentation.append("")
            presentation.append("*These are raw measurements. No interpretation provided.*")
            presentation.append("---")
        
        return "\n".join(presentation)
    
    def present_routine_plan(self, routine: Routine) -> str:
        """
        Presents routine execution plan without persuasion.
        """
        plan = []
        
        plan.append(f"## Routine: {routine.name}")
        plan.append(f"ID: {routine.id}")
        plan.append("")
        plan.append("### Steps")
        
        for i, step in enumerate(routine.steps, 1):
            confirmation = "вң“" if not step.requires_confirmation else "[Requires confirmation]"
            plan.append(f"{i}. {step.agent}.{step.action} {confirmation}")
        
        plan.append("")
        plan.append("### Important")
        plan.append("- No steps will be synthesized or prioritized")
        plan.append("- You may pause or cancel at any step")
        plan.append("- Each step's output will be presented raw")
        plan.append("- I will not suggest modifications or optimizations")
        plan.append("")
        plan.append("Proceed with this exact sequence?")
        
        return "\n".join(plan)
```

```python
    def present_multi_modal_awareness(self, sensor_data: Dict) -> str:
        """
        Presents data from multiple sensors without fusion.
        """
        presentation = []
        
        presentation.append("## Current Environment State")
        presentation.append("*Raw sensor data. No inference or fusion.*")
        presentation.append("")
        
        # Group by sensor type
        for sensor_type, sensors in self._group_by_type(sensor_data).items():
            presentation.append(f"### {sensor_type.replace('_', ' ').title()}")
            
            for sensor_id, data in sensors.items():
                presentation.append(f"**{sensor_id}**:")
                for reading in data['readings']:
                    presentation.append(f"  - {reading['metric']}: {reading['value']} {reading.get('unit', '')}")
            
            presentation.append("")
        
        # Present contradictions if any
        contradictions = self.contradiction_agent.analyze(sensor_data)
        if contradictions:
            presentation.append("### Sensor Contradictions")
            presentation.append("*These are discrepancies, not errors. You decide which is correct.*")
            
            for contradiction in contradictions:
                presentation.append(f"- {contradiction['description']}")
                presentation.append(f"  Sources: {', '.join(contradiction['sources'])}")
                presentation.append(f"  Values: {contradiction['values']}")
            
            presentation.append("")
        
        return "\n".join(presentation)
```

```python
class PrivacyFilterAgent(Agent):
    """
    Applies privacy filters to sensitive data.
    """
    FILTERS = {
        "camera": [
            BlurFacesFilter(),
            RemoveLicensePlatesFilter(),
            BlurTextFilter(),
        ],
        "microphone": [
            RemoveNamesFilter(),
            AnonymizeVoicesFilter(),
        ],
        "location": [
            ReducePrecisionFilter(accuracy=100),  # 100m precision
            RemoveHomeLocationFilter(),
        ]
    }
    
    def apply(self, data: RawData, data_type: str) -> FilteredData:
        """
        Applies filters based on user privacy settings.
        """
        filtered = data.copy()
        
        for filter in self.FILTERS.get(data_type, []):
            if filter.should_apply(data, self.user_privacy_settings):
                filtered = filter.apply(filtered)
        
        return FilteredData(
            data=filtered,
            filters_applied=[f.name for f in applied_filters],
            original_hash=hash(data),  # For audit
            filtered_hash=hash(filtered)
        )
```
