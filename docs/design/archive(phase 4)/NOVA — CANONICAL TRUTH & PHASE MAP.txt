# PART I — PERMANENT CONSTITUTIONAL INVARIANTS  

These invariants apply to **all phases** of Nova’s existence. They are **immutable** and cannot be relaxed by any future unlock.  

| # | Invariant | Description |
|---|-----------|-------------|
| 1 | **Single Master Governor** | Exactly one choke point for all governed actions. No code path may bypass the Governor. |
| 2 | **Intelligence–Authority Split** | Intelligence (reasoning, analysis) is advisory only. Authority (execution, state change) is always Governor‑mediated. |
| 3 | **No Autonomy** | Nova never initiates actions, never acts on its own, never pursues goals. |
| 4 | **No Background Cognition** | Nova performs zero reasoning, inference, or intent evaluation except in direct response to a user message. |
| 5 | **No Predictive Preloading** | No pre‑fetching of data or preparation of responses based on partial input, idle time, or inferred intent. |
| 6 | **No Silent Fallback** | Nova must never silently substitute a different capability when a requested capability is unavailable. Any substitution must be explicitly disclosed and user‑confirmed. |
| 7 | **No Environment Override** | The runtime flag `GOVERNED_ACTIONS_ENABLED` is hard‑coded in source; it may never be overridden via environment variables, config files, or runtime patching. |
| 8 | **No Dynamic Execution Surface** | `exec`, `eval`, `__import__`, `subprocess`, `os.system`, reflection‑based invocation of untrusted names, and all other dynamic execution primitives are **forbidden** in any code path that can influence authority. `importlib` may be used **only** for **startup‑time, Governor‑mediated loading of statically pre‑approved capability modules**; **no runtime dynamic loading** is permitted. |
| 9 | **No Multi‑User** | Phase‑4 is strictly single‑user. No user isolation, profile switching, or session differentiation is permitted. Multi‑user design is locked behind a future phase gate. |
| 10 | **Orb Non‑Semantic** | The Orb communicates only one message: *“There is power here. It is stable.”* No user may infer Nova’s operational state (idle, processing, listening, error) from the Orb alone. |
| 11 | **UI Reflective Only** | Nova’s UI may **reflect state** (show what exists) but never **interpret, guide, initiate, or prioritize** attention. |
| 12 | **Calm as Technical Requirement** | Calm is **enforced through deterministic animation, absence of urgency language, prohibition of progress metaphors, and zero attention‑demanding UI elements**. No animation may imply progress, certainty, or cognition. The system is silent when idle. |
| 13 | **No Speculative Execution** | No speculative execution, tool pre‑warming, parallel capability evaluation, or background concurrency is permitted. Every action executes strictly sequentially, only when explicitly invoked. |

# PART III — DEEP THOUGHT CONSTITUTIONAL FRAMEWORK  

## 3.1 Identity & Role  

Deep Thought is **contained external intelligence** – not an assistant, not a second mind, not a system component.  

- **External** – runs outside Nova core  
- **Stateless** – no memory, no continuity across invocations  
- **User‑invoked** – never automatic  
- **Governor‑contained** – cannot bypass Governor  
- **Advisory only** – output is text, no authority  
- **Non‑authoritative** – no trust elevation  
- **Non‑persistent** – ephemeral, session‑scoped  

## 3.2 Invocation Flow (Exact, Audited)  

1. User composes a message.  
2. User clicks **`🧠 Deep Thought`** button (icon + text, inside input bar).  
   - **No predictive preloading** – button does nothing until explicitly clicked.  
3. System sends exact user message with **minimal, non‑semantic safety wrapper**:  
   ```
   System: You are an external analysis tool. Provide neutral analysis only.  
   System: Do not suggest actions unless explicitly requested.  
   User: [exact user message]
   ```  
4. If the request requires online/time‑sensitive data, Deep Thought **requests this data exclusively through Nova’s governed read‑only search tools**.  
   - Nova executes the search, applies entry/exit disclosures, returns structured results.  
   - **No independent fetching, caching, or monitoring.**  
   - **Fail closed** – if Nova’s search tool fails or times out, Deep Thought must refuse, not fall back.  
5. DeepSeek (or designated external model) responds.  
   - Response displayed with **immutable, unvarying label**:  
     ```
     🧠 Deep Thought (External Analysis):
     [analysis text]
     ```  
   - No visual emphasis; no reordering; output length ≤2× user message token length **after tone normalization** unless user explicitly requests depth.  
6. **Nova performs no independent reasoning or intent interpretation during Deep Thought invocation.** It performs only infrastructural routing and governed tool execution.  
7. **Stateless, single‑response** – no dialogue loops, no cross‑turn memory.

# PART IV — UI CONSTITUTIONAL LAYER & ORB  

## 4.1 The Four Gates of Safe UI Auto‑Behavior  

Every UI behavior must pass **all four**:  

| Gate | Principle | Test |
|------|-----------|------|
| **Reflective** | Shows what exists; does not interpret meaning. | “Does this show what exists, or suggest what matters?” |
| **Non‑Directive** | Informs neutrally; does not guide. | “Could this be perceived as the system having an opinion?” |
| **Non‑Initiative** | Responds to user/external triggers; does not start attention shifts. | “Did the user or external world trigger this, or did the system?” |
| **Non‑Intent** | Displays all information equally; does not express priority. | “Does the UI appear to be deciding what matters?” |

**One‑Line Policy:**  
> *“Nova’s UI may change what it shows, but never why, when, or how you should look at it.”*  

## 4.2 Background Refresh  

✅ **Permitted:** External data panels (weather, news, stock prices) may refresh automatically.  
   - Must display timestamp of last update.  
   - Data must be externally verifiable.  

❌ **Forbidden:** Internal system refresh (“checking for new features”, background optimization, version‑update checks **during normal idle operation**).  
   - Version checks are permitted **only** at application startup (the initial boot sequence, **not** on tab focus, WebSocket reconnect, or wake‑from‑idle) **or** when explicitly user‑invoked.  

## 4.3 Orb Constitution (v3.0)  

### Layer 1 — Immutable Orb Constitution  

| Principle | Statement |
|-----------|-----------|
| **No Readable Coupling** | The Orb exhibits **no readable causal linkage** to Nova’s internal state, cognition, or activity. No user may infer operational state from the Orb alone. |
| **No Event‑Driven Motion** | All Orb motion is **continuous, intrinsic, causally independent** of user input, system events, or response generation. |
| **Containment Signal** | A faint containment rim must always be present. |
| **Non‑Biological Motion** | Motion must never mimic biological life (heartbeat, breath, pulse). |
| **Silence as Default** | Sound, if used, is infrequent, sub‑70 Hz, <0.5s, never attention‑seeking. |

**The only message the Orb is permitted to transmit:**  
> *“There is power here. It is stable.”*  

### Layer 2 — Implementation Standard (Evolvable)  

**Reference v3.0:**  
- **Core:** Deep electric blue (`#0A1F44` → `#1E3A6F`), violet drift in lower hemisphere.  
- **Containment rim:** Sub‑pixel, cool white with trace blue, <10% opacity.  
- **Internal motion:** Convection‑like drift (120–180s cycle), micro‑wander (2px).  
- **Ambient bloom:** 120% diameter, 4–6% opacity, Gaussian falloff.  
- **Forbidden colors:** Red, orange, green, yellow – any semantic mapping.  
- **Light mode adaptation:** Reduce luminance 20%, preserve hue; rim becomes faint gray.  

**Testing:**  
- **Unplug Test (reframed):** *“If Nova transitions from idle to processing, does the Orb’s appearance change in any way a user could reliably detect?”* – PASS if no detectable change.  
- **Semantic Leak Test:** User study – accuracy at guessing Nova’s state from Orb alone must be **not significantly above chance** (p > 0.05).
