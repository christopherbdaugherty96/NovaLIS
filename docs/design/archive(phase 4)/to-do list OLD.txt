## 🔉 5. WAKE WORD GOVERNANCE

### 5.1 Constitutional Permission

**Wake word is allowed because:**
- It's a local pattern matcher, not cognition
- It requires explicit user opt-in
- It provides no additional authority
- It's fully inspectable and reversible

### 5.2 Status & Constraints

- **Default:** DISABLED (Phase 3.5)
- **Opt-in only:** User must explicitly enable via Safety Dashboard
- **Local activation only:** No cloud dependency
- **Wake word ≠ background cognition**

### 5.3 Audio Processing Rules

**Pre-wake:**
- Pattern matching only
- No transcription
- No storage
- Audio discarded immediately after detection

**Post-wake:**
- Real-time streaming to STT only
- No buffering beyond immediate needs
- Session boundary: All audio state destroyed at conversation end

**Storage & Logging:**
```
/nova_state/wake_word/
├── activations.jsonl     # Append-only: timestamp, confidence
├── settings.json        # User preferences (enabled/disabled, timeout)
└── NEVER stores audio files or transcripts
```

### 5.4 Activation Flow

```
[IDLE State]
    ↓
[Wake Word Detected] → Audio sample discarded after detection
    ↓
[LISTENING State] → Orb: LISTENING, status: "Listening..."
    ↓
[Audio → STT] → Real-time transcription only
    ↓
[Processing] → Orb: PROCESSING
    ↓
[Response + Continue Listening] → Returns to LISTENING
    ↓
[3s silence OR "Stop listening"] → Returns to IDLE
```

**Timeout Configuration:**
- Default: 3 seconds silence
- User configurable: 1-10 seconds
- No adaptive timeouts

### 5.5 Privacy Safeguards

- Audio buffer cleared immediately after transcription
- No cross-session audio retention
- Wake word model runs in isolated process
- User can review activation log and clear at any time
- Circuit breaker: Max 5 activations/minute → auto-pause with notification
