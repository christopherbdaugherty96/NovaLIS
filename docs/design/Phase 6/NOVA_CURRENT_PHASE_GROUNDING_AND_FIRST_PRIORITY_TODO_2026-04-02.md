# Nova Current Phase Grounding And First Priority TODO
Date: 2026-04-02
Status: First-priority grounding packet
Scope: Record the clearest current answer to "what phase is Nova actually in?" and lock the first roadmap priority that should guide near-term work

## Why This Packet Exists

Nova now spans multiple phase layers at once.

That is a sign of real progress, but it also creates confusion:
- older early-phase foundations are still active
- middle-phase trust and reasoning work is already complete
- later-phase execution work has started but is not finished
- product-direction docs can make Nova look further ahead than the runtime really is

This packet exists to save one clear grounding statement in the roadmap itself.

## The Honest Current Phase Status

Nova is not cleanly "in one phase."

The current runtime truth is:
- Phase 3.5: complete
- Phase 4: active
- Phase 4.2: active
- Phase 4.5: partial
- Phase 5: active and effectively closed as a real runtime layer
- Phase 6: complete
- Phase 7: complete
- Phase 8: active, but still narrow

The simplest honest summary is:

`Nova is currently a Phase-7-complete system with partial Phase-8 execution foundations and unfinished Phase-4.5 polish.`

That means:
- the trust spine is beyond Phase 6
- the reasoning and research layer is beyond Phase 7
- the action and operator layer is only at the beginning of Phase 8
- the user-friendliness and home-assistant feel still need more Phase-4.5 completion work

## What Nova Already Has

Nova already has a real runtime with:
- governed web research
- answer-first news and summary flows
- multi-source reporting
- screen capture and explanation
- governed memory
- continuity surfaces
- trust and settings surfaces
- bounded external reasoning review
- live dashboard, websocket, and voice layers

Nova also already has:
- a real Governor spine
- a real capability registry
- real audit and execution boundaries
- a narrow OpenClaw home-agent foundation

That means Nova is already real as:
- a governed intelligence workspace
- a research and explanation assistant
- a continuity and memory assistant
- a narrow operator surface

## What Nova Is Not Yet

Nova is not yet:
- a fully connector-rich assistant
- a full broad operator across apps and sites
- a generally available governed automation platform
- a complete Phase-8 governed envelope execution system

The main runtime gaps remain:
- proper calendar integration
- full Phase-8 governed envelope execution
- final user-facing polish on the still-partial Phase-4.5 layer

## First Priority Truth

The first near-term priority should not be:
- Phase 9 expansion
- Phase 10 adaptation theory
- more future-lane sprawl

The first priority should be:

`Finish the current product foundation before widening Nova further.`

That means locking attention on three things:
1. finish Phase-4.5 polish where the product still feels partial
2. finish the highest-value practical usefulness gap through real connectors
3. finish the missing full Phase-8 governed execution foundation

## The Correct Next Order

### Priority 1 - Finish product-foundation polish
This is the true first priority.

Do next:
- live-device TTS confidence validation
- real manual feel-pass on the updated UX
- setup and readiness clarity cleanup
- screen-help smoothness and first-use success
- continued reduction of technical wording in user-facing surfaces

Reason:
- the runtime itself still marks Phase 4.5 as partial
- Nova should feel calm and solid before it feels broader

### Priority 2 - Widen safe usefulness through connectors
Best first connector:
- real calendar integration

Then:
- one or two other high-value read-heavy connectors

Reason:
- this is the most obvious usefulness gap in the current runtime
- it gives Nova more real-life value without weakening the trust model

### Priority 3 - Finish full Phase-8 governed execution foundations
This means:
- move beyond only the narrow OpenClaw home-agent foundation
- complete the broader safe execution layer
- preserve explicit budgets, previews, stop controls, and trust visibility

Reason:
- this is the real architectural gap between Nova as a narrow assistant and Nova as a governed operator

## What This Packet Blocks

This grounding should block the following mistakes:
- pretending Nova is already in Phase 9 or 10 implementation territory
- treating future product-lane docs as current build priority
- widening autonomy before current trust and UX surfaces feel finished
- confusing OpenClaw foundation work with full governed execution completion

## Anchor Statement

If a single sentence is needed for roadmap grounding, use this:

`Nova stands on a strong Phase-5-through-7 runtime base, with Phase 8 started but not finished, and Phase-4.5 polish still needing completion.`

That is the current truth the roadmap should optimize around.
