# Phase-6 Local AI Appliance and Product Direction
Date: 2026-03-13
Status: Product-direction note only; not runtime truth
Scope: Nova Hub / local AI appliance framing for the Phase-6 era and beyond

## Purpose
This document explains what Nova would look like as a real local AI product or appliance and how that direction fits the architecture already built.

## Hardware Concept
A consumer Nova device could be framed as a local AI node:

`Nova Hub`

A small appliance with:
- local AI processing
- microphones
- speaker
- calm orb interface
- no ads
- no surveillance-first business model

## Example Hardware Profile
| Component | Example |
| --- | --- |
| CPU | Apple Silicon / AMD APU / ARM |
| RAM | 16-32 GB |
| Storage | 1 TB |
| Mic array | wake-word detection |
| Speaker | Nova voice responses |
| LED orb | Nova presence |
| Connectivity | Wi-Fi / Ethernet |

## Core Differentiation
Most mainstream assistant devices are cloud-first.

Nova's differentiator is:
- local plus governed AI
- Governor over intelligence
- ledger logging
- explicit invocation
- no hidden background autonomy
- privacy-first architecture

## Three Sellable Core Features

### 1. Explain anything on my screen
The strongest signature interaction is:
- `Nova what is this?`

This works on:
- webpages
- PDFs
- spreadsheets
- contracts
- code
- error messages

This is described in:
- `docs/design/Phase 6/PHASE_6_PROGRESSIVE_SCREEN_INTELLIGENCE_PRODUCT_SPEC.md`

### 2. Local research assistant
Example:
- `Nova research the best electric cars under $40k`

Nova can produce structured research briefs because it already has:
- multi-source reporting
- structured renderer paths
- verification / intelligence flows

### 3. Governed computer control assistant
Examples:
- `Nova open the Nova project`
- `Nova open my spreadsheet`
- `Nova create a research brief`

Nova is well suited to this because it already has:
- governed actions
- capability registry
- execution boundary
- ledger traceability

## Everyday Consumer Experience
The desired feel is:
1. plug in Nova
2. connect to Wi-Fi
3. start interacting immediately

Daily loop examples:
- `Nova what is this?`
- `Nova summarize this document`
- `Nova explain this error`
- `Nova research this topic`
- `Nova open my spreadsheet`

## Productization Path
For the current repository state, the practical path is:

### 1. Keep the Phase-5 product layer stable
Phase 5 is already closed.
That means the right move is to preserve and polish:
- memory
- continuity
- tone controls
- scheduling
- pattern review

### 2. Make progressive screen intelligence a signature surface
This is the most understandable "magic moment" feature.

### 3. Strengthen file and document intelligence
This is high-value daily utility:
- reports
- PDFs
- contracts
- spreadsheets

### 4. Improve installability
Real product readiness requires:
- one-click install or close to it
- startup readiness diagnostics
- clearer model / network / microphone status

### 4.5. Package Nova as real desktop software
Recommended first distribution model:
- local desktop app

Recommended early channels:
- GitHub Releases
- direct download website
- technical early-adopter communities

### 5. Package toward a local AI hub
Only after the software experience is strong enough should Nova be treated as an appliance package direction.
