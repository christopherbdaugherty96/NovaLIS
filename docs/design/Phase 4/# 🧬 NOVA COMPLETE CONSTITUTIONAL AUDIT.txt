1. **Orb Testing:** The blueprint mentions “Semantic Leak Test” but omits the “Unplug Test” refinement (which was reframed in `Nova Orb.txt`). However, the essence of that test is already captured by the “No Readable Coupling” principle and the semantic test. This is a harmless omission.
2. **Personality Agent Validation Pipeline:** The four validators are named but not detailed. This is appropriate for a constitutional summary; detailed design remains in the orthogonal cognition stack document.
3. **Mutation Firewall Pipeline:** The 9 stages are listed, but the exact ordering (e.g., Stage 8 “Simulate & Verify” before commit) is implied rather than enumerated. The source document’s lifecycle diagram is more detailed, but the constitutional summary suffices.
4. **Tier 4 Addendum:** The blueprint states that background cognition is permitted “only when all of the following hold” and lists six conditions. This matches the addendum in v2.1. Good.
5. **Multi‑User Phase Gate:** Invariant #17 correctly notes that multi‑user is not supported in Phase 4 and requires future unlock. This aligns with v2.1’s reclassification as Tier 2.

None of these observations represent errors or contradictions; they are merely differences in depth of detail between a summary and a full design document. The blueprint correctly maintains all constitutional boundaries.
