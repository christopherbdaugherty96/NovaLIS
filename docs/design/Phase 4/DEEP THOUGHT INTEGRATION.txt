## 🏛️ PART I – CANONICAL ARCHITECTURE

### 1.1 Invocation Flow (Exact, Audited, Stateless)

1. **User composes a message** in Nova chat input.
2. **User clicks `🧠 Deep Thought` button**  
   - Button: icon + text, placed inside input bar, visually distinct from Send.
   - Active state indicates processing.
   - **No predictive preloading:** Button does nothing until explicitly clicked. No pre‑fetching based on partial input, idle time, or contextual inference.
3. **System routes message with a minimal, non‑semantic safety wrapper** – no augmentation of user intent, no contextual injection, no hidden prompt engineering.  

   **Permitted wrapper (exact form):**
   ```
   System: You are an external analysis tool. Provide neutral analysis only.  
   System: Do not suggest actions unless explicitly requested.  
   User: [exact user message]
   ```
4. **If the user’s message requires online/time‑sensitive data**, Deep Thought **requests this data exclusively through Nova’s governed read‑only search tools**.  
   - Nova executes the search, applies entry/exit disclosures (when applicable), and returns structured results.
   - Deep Thought may **not** chain multiple search tool invocations within a single request **unless explicitly required** by the user’s question. One request → bounded retrieval.
   - If Nova’s read‑only search tool fails or times out, Deep Thought **must fail closed** — no fallback retrieval, no alternative source.
   - All logging, citation, and source validation follow Nova’s existing policies.
5. **DeepSeek (or designated external model) responds.**  
   Response is displayed **immediately** with immutable, unvarying label:

   ```
   🧠 Deep Thought (External Analysis):
   [analysis text]
   ```
   - Output label **never** receives visual emphasis (color, size, animation) beyond standard chat text.
   - Output position is **chronological**, never reordered or prioritized.
6. **Nova is operationally dormant at the reasoning layer during Deep Thought invocation.**  
   - Permitted: storage, logging, routing, executing requested read‑only searches.  
   - **Forbidden:** pre‑fetching tools, evaluating intents, pre‑computing execution pathways, parallel reasoning, intent interpretation, or any cognitive processing beyond infrastructural necessity.
7. **Deep Thought is single‑response and stateless.**  
   - No follow‑up questions that create a dialogue loop.  
   - If user’s message is ambiguous, Deep Thought may **clarify only within the same analysis block** (e.g., “If you meant X: … If you meant Y: …”).  
   - **No conversation ID persistence across invocations.** Each call is independent; no cross‑turn memory.

### 1.5 UI Constitutional Principles for Nova & Deep Thought

- **Allowed:** Displaying current state, external data, system status.
- **Forbidden:** Analyzing, inferring, or interpreting meaning.
- **Test:** “Does this show what exists, or suggest what matters?”

#### Gate 2 – Non-Directive

- **Allowed:** Showing all information equally.
- **Forbidden:** Expressing priority, importance, or relevance judgments.
- **Test:** “Does the UI appear to be deciding what matters?”

**One-Line UI Policy:**
> **“Nova’s UI may change what it shows, but never why, when, or how you should look at it.”**
