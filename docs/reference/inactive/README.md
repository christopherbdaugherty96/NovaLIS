# Inactive Reference Packets
Updated: 2026-04-12
Status: Inactive reference set

## Purpose
This folder stores useful rule packets, checklists, and build standards that are worth preserving but are not active Nova-wide requirements.

These packets are:
- reference material
- optional standards for specific project types
- non-authoritative unless explicitly adopted into an active guide or build rule

## How To Use This Folder
Use this folder when you want to keep a strong external-standard packet available for later adoption without accidentally turning it into a universal Nova rule.

Typical examples:
- framework-specific build rules
- storefront standards
- external deployment checklists
- specialized QA packets for ecosystems Nova is not actively standardizing today

## Authority Note
Documents in this folder do not override:
- `docs/current_runtime/`
- active Nova human guides
- active Nova design authority

If one of these packets is later adopted, it should be copied or promoted into the correct active docs surface with explicit wording.
