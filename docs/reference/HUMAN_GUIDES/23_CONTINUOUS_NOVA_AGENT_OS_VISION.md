# Continuous Nova Agent OS Vision
Updated: 2026-03-21
Status: Human guide

## What This Guide Is
This is the plain-language version of the bigger Nova vision.

It explains what Nova is gradually turning into if you keep building in the current direction.

## The Short Version
Nova is not just becoming a chatbot.

Nova is becoming a **personal agent system**.

That means the long-term goal is for Nova to be able to:
- talk naturally
- help with projects
- work with code and documents
- connect to apps and sources
- remember important things you asked it to remember
- help with local and home systems
- stay available over time
- keep one stable personality
- eventually adapt carefully

But the key part is:

Nova should do all of that **without losing trust**.

## What Kind Of System This Really Is
The cleanest way to think about Nova is:

**one assistant identity, many subsystems, one trust model**

That means:
- Nova should feel like one assistant
- the work underneath can be made of many layers
- all of those layers should follow the same rules about approval, safety, and user control

## What The Main Layers Are

### 1. Conversation
Nova should:
- understand the topic
- follow the thread
- ask better clarifying questions
- explain things clearly

### 2. Project Work
Nova should:
- understand the repo or workspace
- help with docs and reports
- work with GitHub and coding tasks
- stay aware of the current project

### 3. Memory
Nova should:
- let you save things
- remember explicit notes
- show saved memories
- let you edit and delete them
- use them later when relevant

### 4. Connectors
Nova should eventually connect to the tools you actually use, like:
- GitHub
- text files
- Word docs
- Google Docs
- Notion
- Obsidian
- coding workspaces
- later, other apps and systems

### 5. Actions
Nova should eventually be able to:
- open things
- update things
- help with local system actions
- help with home actions later

But only under clear trust and approval rules.

### 6. Always-On Tasks
Nova should eventually support:
- schedules
- reminders
- daily brief
- project refreshes
- recurring checks

This is part of what makes Nova feel continuously available.

### 7. Trust
Nova must keep:
- clear permissions
- clear approvals
- clear logs
- clear boundaries

This is the part that matters most if Nova becomes more powerful.

### 8. Learning
Much later, Nova may be able to:
- adapt
- refine preferences
- get better over time

But this should not come before:
- review
- rollback
- user control

## What "Continuously On" Should Mean
It should not mean:
- secret actions
- hidden learning
- Nova doing whatever it wants

It should mean:
- always available
- able to continue work
- aware of your current projects
- able to use saved memory
- able to run your approved recurring tasks
- still under the same trust rules

## The Right Build Order
If you want Nova to become this system, the safest order is:

### First
- project/workspaces
- memory
- connectors
- better search/news/research
- conversation quality
- evals and traces

### Then
- schedules
- task center
- stronger trust/permission surfaces
- read-only external connectors

### Then
- governed write/update actions
- GitHub/document updates
- app actions
- home integration actions

### Much Later
- learning
- adaptive behavior
- personality refinement
- maybe limited autonomy in bounded areas

## Bottom Line
The goal is not to make Nova a giant uncontrolled super-agent.

The goal is to make Nova:
- continuously useful
- project-aware
- connector-aware
- memory-aware
- trustworthy
- increasingly capable

That is what would make Nova feel like a real personal agent system instead of a collection of features.
