# 🔄 Nova System Flow  
*End-to-End Data Flow of the Entire System*

---

# 1. Chat Flow (User → AI → UI)

User types message
→ dashboard.js sends WS message
→ brain_server receives it
→ nova_tools checks for skills
→ If skill matches → return skill_response
→ Else send to Ollama
→ Ollama streams tokens
→ brain_server relays chunks
→ dashboard.js displays tokens in real-time
→ ORB state changes (thinking → speaking → idle)
→ Memory stored

# 6. Error Handling

- Invalid WS messages → ignored  
- Skill exceptions → fallback AI response  
- Ollama failure → Retry mechanism  
- Missing skill → fallback AI response
