# 🎨 Nova Frontend Dashboard — Technical Overview

---

# ⭐ Purpose
The dashboard is Nova’s **face**, providing:
- Visual ORB interface  
- Real-time chat  
- System widgets  
- Voice controls (future)  

---

# 📁 File Overview

### **index.html**
Defines the layout:
- Left: ORB  
- Center: Chat  
- Right: Widgets  

### **dashboard.js**
Handles:
- WebSocket connection  
- Chat message rendering  
- Streaming text  
- Widget updates  
- ORB state syncing  

### **orb.js**
A custom 3D renderer for the glowing ORB.

### **style.css**
High-end cyberpunk/glassmorphism theme.

---

# 🔌 WebSocket Message Types

Frontend receives:
- `chat_chunk`  
- `chat_done`  
- `skill_response`  
- `widget_update`  
- `orb_state`  

Frontend sends:
- `chat_message`  
- (future) `audio_chunk`  

---

# 🟣 ORB Behavior
States:
- idle  
- thinking  
- speaking  
- error  
- sleeping  

Uses particle and wave effects for emotion simulation.

---

# 📱 Responsiveness
Breakpoints:
- Desktop  
- Tablet  
- Mobile  

All widgets reorganize dynamically.

---

The frontend is visually expressive, responsive, and real-time reactive.

# 🧩 Nova Skills System — Complete Guide

---

# ⭐ What Are Skills?

Skills are **modular Python functions** that extend Nova’s abilities.

They can:
- Return structured data  
- Trigger widget updates  
- Act autonomously (if scheduled)  
- Be added/removed easily  

---

# 📁 Skills Structure

skills/
├── weather.py
├── news.py
├── system.py
└── files.py

yaml
Copy code

---

# 🏗️ Skill Template

```python
def skill_example(param="test"):
    return {
        "success": True,
        "data": {...},
        "widget_data": {...}
    }

def register(registry):
    registry.register_skill("example", skill_example)
